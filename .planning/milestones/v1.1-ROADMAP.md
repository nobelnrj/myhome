# Milestone v1.1: Accounts, Assets & Household Polish

**Status:** ✅ SHIPPED 2026-06-20
**Phases:** 8-12 (incl. inserted 11.1)
**Total Plans:** 26

## Overview

v1.1 grew My Home from an automated expense tracker into a light household finance + ops hub. It opened with a stabilization pass (crash fixes, category ordering, routine-reset scaffold), then delivered an additive SchemaV6→V9 evolution carrying Accounts, Assets, transfer/routine fields, and a RoutineCompletion model. On top of that schema spine it shipped full Accounts management with live balances and per-account spend, deterministic self-transfer detection with an explicit confirm flow, a net-worth Asset Tracker (manual holdings + free AMFI MF NAV + allocation/trend charts), SIP automation with NPS NAV auto-refresh, and a Notes daily-routine enhancement (calendar surfacing, timed notifications, drag-reorder, streak/history). All work stayed local-only, free-data-only, with manual override always available.

## Milestone Summary

**Decimal Phases:**

- Phase 11.1: SIP Automation and NPS NAV auto-refresh (inserted after Phase 11)

**Key Decisions:**

- Additive-only schema evolution (V5→V9) with all typealiases flipped together each bump — the schema-typealias footgun (STAB-08) forced a discipline of flipping every typealias atomically.
- Self-transfer detection is auto-detect + mandatory confirm, never silent exclusion.
- AccountBalance sign convention fixed: spends stored POSITIVE, balance = baseline − net spend (inverted through Phase 9, corrected in Phase 10).
- Asset prices from free public sources only (AMFI reliable; stock auto-fetch deferred), manual override always available.
- Daily-routine reset implemented at note level (`routineLastResetDate`) in Phase 9; per-block `RoutineCompletion` model added in Phase 12 for streak/history.
- STAB-03 reversed mid-execution to insert new categories at the TOP per user direction (supersedes original "bottom" decision D-05).

**Issues Resolved:**

- Two production crash vectors (Notes calendar tombstone access; Gmail sync Category-after-await + per-loop save).
- Schema typealias mismatch crash (Note/NoteBlock left at SchemaV4 while container on V5).
- Expenses tab badge stuck with no actionable row (badge now derived from actionable transfer pairs).
