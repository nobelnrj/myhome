## Spacing Scale

SwiftUI uses a point-based coordinate system. All values are in pts (1pt = 1px on non-Retina, 2px on Retina @2x, 3px on Retina @3x). The 8-point grid maps directly to pt values.

Declared values (multiples of 4):

| Token | Value | SwiftUI Expression | Usage |
|-------|-------|--------------------|-------|
| xs | 4pt | `.padding(4)` / `spacing: 4` | Icon gaps, inline label padding |
| sm | 8pt | `.padding(8)` / `spacing: 8` | Compact element spacing, row internal gaps |
| md | 16pt | `.padding(16)` / `spacing: 16` | Default section padding, form field spacing |
| lg | 24pt | `.padding(24)` / `spacing: 24` | Card internal padding, section breaks |
| xl | 32pt | `.padding(32)` / `spacing: 32` | Screen-level horizontal margins (use `.padding(.horizontal, 32)`) |
| 2xl | 48pt | `spacing: 48` | Major vertical section separators |
| 3xl | 64pt | `spacing: 64` | Full-bleed hero areas (not used in Phase 1) |

Exceptions:
- Minimum tappable touch target: 44pt × 44pt (Apple HIG minimum). All interactive controls (delete button, edit row tap area, keypad keys) must meet this minimum. Source: Apple HIG.
- Amount keypad keys: 64pt × 64pt (comfortable for primary interaction path per EXP-01 ≤4-tap rule). Source: CONTEXT.md D-04 (standard decimal keypad).
- Navigation bar back button: system-managed (do not override).

## Typography

Uses SwiftUI `.font()` semantic styles mapped to SF Pro. Do not use explicit `size:` overrides — use semantic roles so Dynamic Type scales automatically (accessibility requirement for iOS apps).

Exactly 4 font-size roles are declared:

| Role | SwiftUI Style | Approx Size (default) | Weight | Line Height | Usage |
|------|---------------|-----------------------|--------|-------------|-------|
| Display | `.font(.largeTitle)` | 34pt | Bold (700) | 1.2 | Amount field in add/edit sheet — the single focal number |
| Heading | `.font(.title2)` | 22pt | Semibold (600) | 1.2 | Screen titles (NavigationStack inline title), section headers |
| Body | `.font(.body)` | 17pt | Regular (400) | 1.5 | Expense list row primary text (note/memo, date) |
| Label | `.font(.subheadline)` | 15pt | Regular (400) | 1.4 | Secondary row text (date string in list), form field labels, timestamp detail, currency-code badge |

Weight summary: Regular (400) + Semibold/Bold (600/700) — two functional weights only, consistent with HIG.

Note: Caption (`.caption`, ~12pt) is intentionally omitted. Timestamp detail and currency-code badge use the Label role (`.subheadline`, 15pt) — sufficient contrast hierarchy at this scale and preserves Dynamic Type readability.

Dynamic Type: all styles must respond to accessibility size categories. Never use fixed frame heights that clip text.

## Color

This is an iOS 17+ app. Colors use SwiftUI semantic system colors (adaptive light/dark mode automatically). No hardcoded hex values in v1 — use system semantic names. Explicit hex values below are the light-mode reference values for documentation only.

| Role | SwiftUI Color | Light Hex (reference) | Dark Hex (reference) | Usage % | Usage |
|------|---------------|-----------------------|----------------------|---------|-------|
| Dominant (60%) | `.systemBackground` | `#FFFFFF` | `#000000` | 60% | App background, list background, sheet background |
| Secondary (30%) | `.secondarySystemBackground` | `#F2F2F7` | `#1C1C1E` | 30% | List row grouped background, form section background, keypad container |
| Accent (10%) | `.systemIndigo` (mapped to `Color.accentColor`) | `#5856D6` | `#7B79E6` | 10% | See reserved-for list below |
| Destructive | `.systemRed` | `#FF3B30` | `#FF453A` | On-demand | Delete expense confirmation button only |
| Positive amount | `.systemGreen` | `#34C759` | `#30D158` | Inline | Negative-amount entries (refunds) displayed in green to signal credit |
| Separator | `.separator` | `#C6C6C8` | `#38383A` | Structural | List row dividers (system-managed via List) |

**Accent reserved for (explicit list):**
1. "Add Expense" primary CTA button (filled, in toolbar and empty state)
2. Amount field cursor and active ring
3. "Save Expense" button in NavigationStack toolbar
4. Keypad "confirm amount" submit action (if distinct from Save Expense)
5. Tappable date in expense row (if tappable to filter — Phase 2+; reserve now)

Accent is NOT used for: list row backgrounds, destructive actions, informational text, icons that are decorative.

## Screen Inventory (Phase 1 only)

Three screens ship in Phase 1. All other screens (Categories, Budget, Notes, Overview, Settings, Face ID) are out of scope.

### Screen 1: Expense List

**Navigation:** Root tab or root NavigationStack view (single screen, no tabs in Phase 1).
**Purpose:** Show all expenses reverse-chronologically. Entry point for add, edit, delete.

Layout contract:
- `NavigationStack` with `.navigationTitle("Expenses")` in `.inline` display mode.
- `List` with `.listStyle(.insetGrouped)`.
- Each row: leading = amount (Display weight, right-aligned in a fixed 100pt column), trailing = note text (Body, one line, truncated with `…`) + date (Label, `.subheadline`, secondary color).
- Negative amounts rendered in `.systemGreen` accent; positive in `.label` (default text color).
- Amount formatted with `Locale(identifier: "en_IN")`: `₹1,00,000.00`. Source: CONTEXT.md D-04, FND-07.
- Swipe-to-delete on each row (`.onDelete` handler). Swipe action label: "Delete" in `.systemRed`.
- Tap row to open edit sheet.
- Toolbar trailing button: `+` (SF Symbol `plus`) with `.systemIndigo` accent — opens add sheet.
- Empty state: centered placeholder (see Copywriting Contract).

### Screen 2: Add Expense Sheet

**Navigation:** Presented as `.sheet` from list toolbar `+` button.
**Purpose:** Capture amount, optional note, optional date override. ≤4 taps to save. Source: CONTEXT.md D-01, D-04, EXP-01.

Layout contract:
- Sheet presented with `NavigationStack` inside for toolbar buttons.
- `.navigationTitle("New Expense")` in `.inline` mode.
- Toolbar: leading = "Cancel" (platform-standard iOS sheet-dismiss control; see Copywriting Contract), trailing = "Save Expense" (`.systemIndigo`, disabled until amount > 0).
- Section 1 — Amount (full-width):
  - Large centered amount display using `.font(.largeTitle)`, `.fontWeight(.bold)`.
  - Amount rendered live as user types: `₹0.00` placeholder, updates to `₹{amount}` with en-IN formatting.
  - Negative toggle: a `-/+` toggle button (SF Symbol `plus.slash.minus`) at left of amount display. Tapping flips sign. Source: CONTEXT.md D-03.
  - Custom decimal keypad below amount (0–9, decimal point, backspace). Keys are 64pt × 64pt minimum, laid out in 3-column grid. NOT system keyboard (avoids keyboard avoidance layout thrash and satisfies the "keypad" UX in EXP-01). Source: CONTEXT.md D-04.
- Section 2 — Optional fields (collapsed by default to keep ≤4-tap path):
  - Date picker row: label "Date", value defaults to now (formatted as "Today, 9:41 AM" if today, else "28 May 2026, 9:41 AM"). Tapping expands an inline `DatePicker`. Source: CONTEXT.md D-01, D-02.
  - Note field row: label "Note", `TextField` single-line, placeholder "Merchant or memo". Source: CONTEXT.md D-05 (`note: String?`).
- No category picker in Phase 1. Source: CONTEXT.md D-07.
- Save Expense is enabled when amount is non-zero (positive or negative). Zero is invalid.

### Screen 3: Edit Expense Sheet

**Navigation:** Presented as `.sheet` when user taps an existing expense row.
**Purpose:** Edit amount, note, date. Delete from this screen.

Layout contract:
- Identical form layout to Add Expense Sheet.
- `.navigationTitle("Edit Expense")`.
- Toolbar: leading = "Cancel" (platform-standard iOS sheet-dismiss control; see Copywriting Contract), trailing = "Save Expense" (active when any field changed).
- Additional: destructive "Delete Expense" button at bottom of form, outside sections — full-width, `.tint(.systemRed)`, role `.destructive`. Source: EXP-03.
- Delete triggers confirmation action sheet (not alert) with two actions: "Delete Expense" (destructive) and "Cancel" (standard action-sheet dismiss). Source: Copywriting Contract below.

## Interaction Contracts

### ≤4-Tap Add Flow (hard requirement EXP-01)

Tap 1: Tap `+` toolbar button → sheet opens, cursor in amount field, keypad visible immediately (no keyboard show animation needed — custom keypad always visible).
Tap 2: Type amount via keypad (multiple key taps count as one "interaction step" per EXP-01 — the requirement counts screen transitions, not keystrokes).
Tap 3: Tap "Save Expense" → expense saved, sheet dismisses, new row appears at top of list.

Optional extras (not in the ≤4-tap critical path):
- Tap date row → expand inline date picker → change date.
- Tap note field → type memo.
- Tap `-/+` → flip sign.

### Swipe-to-Delete

Standard SwiftUI `.onDelete` with `.listStyle(.insetGrouped)`. Swipe left to reveal red "Delete" label. Confirm via the revealed button — no secondary confirmation at list level (confirmation only in Edit sheet's full-delete flow). Rationale: swipe delete on a list is low-risk discovery; the edit-sheet delete button is the primary delete path.

### Edit Navigation

Tap any expense row → Edit sheet slides up. All fields pre-filled from the tapped `Expense` model. Save Expense button disabled until a field is changed (`isDirty` state flag). Cancel discards changes with no confirmation (standard iOS sheet-dismiss pattern).

## Accessibility Contracts

- All interactive elements have explicit `.accessibilityLabel(_:)` where the visual label is icon-only (e.g., `+` button, `-/+` toggle).
- Custom keypad keys expose `.accessibilityLabel("0")` through `.accessibilityLabel("9")`, `.accessibilityLabel("Decimal point")`, `.accessibilityLabel("Delete")`.
- Amount display reads full formatted string: `accessibilityValue("₹1,234.50")`.
- Minimum touch target 44pt × 44pt enforced via `.frame(minWidth: 44, minHeight: 44)` where needed.
- Color is never the only differentiator: negative amounts show green color AND a leading minus sign in the formatted string (`-₹500.00`).
- Dynamic Type: no fixed-height containers that clip text.
