## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input -> Expense.amount | Untrusted keypad/text input parsed to Decimal crosses into persistence here |
| user input -> Expense.note | Free-form text stored and later rendered |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03 | Tampering | AddExpenseView amount parse | mitigate | Parse keypad String to Decimal with a guard; reject non-numeric and enforce abs(amount) < 1_000_000_000 before insert (ASVS V5 input validation; RESEARCH Security Domain). Keypad already constrains to digits + one dot + 2 decimals. |
| T-01-06 | Tampering | Expense.note rendering | mitigate | Render note only via SwiftUI Text() (plain text, no HTML). Do NOT use AttributedString(markdown:) on user input (RESEARCH Security Domain). |
| T-01-07 | Denial of Service | sheet dismissal / navigation race | mitigate | Dismiss sheets cleanly after save/delete; never mutate selection in the same tick as a navigation change (Pitfall 19). |
| T-01-SC | Tampering | npm/pip/cargo installs | mitigate | Not applicable - zero third-party packages this phase (RESEARCH Package Legitimacy Audit); all Apple frameworks. |
</threat_model>
