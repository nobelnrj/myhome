<objective>
Create the closed `PantryCategory` enum and the single, total `category ‚Üí (SF Symbol, DesignTokens colour)` mapping table that ALL icon rendering flows through ‚Äî the keyword fallback today, the on-device model in 22-02/22-03.

This is the structural mechanism behind ICON-02: because the mapping is a Swift `switch` over a closed enum, no caller ‚Äî least of all a language model ‚Äî can express an SF Symbol name at all. The 20-03 empty-tile bug (`takeoutbag.fill.and.rectangle.portrait`, a plausible-but-fake symbol that rendered nothing and raised no error, and that a passing unit test did not catch) is designed out rather than tested for.

Purpose: give 22-02 and 22-03 a stable contract to build against, with the existing keyword behaviour provably unchanged.
Output: `PantryCategory.swift`, a refactored `KitchenLogic` whose keyword rules yield categories rather than raw symbol strings, and a test suite pinning mapping totality plus zero visual regression.
</objective>
