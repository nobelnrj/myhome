<locked_decisions>
Resolved from AI-SPEC §6, binding:

- **P22-D2 (two-enum mirror):** `@Generable` cannot be applied conditionally and drags `@available(iOS 26, *)` through every type that mentions it. So the model-facing enum `GeneratedPantryCategory` is a SEPARATE iOS-26-only twin that maps 1:1 onto the plain `PantryCategory` from 22-01. A parity test pins the two case sets together, so adding a category to one and forgetting the other is a test failure rather than a silently unreachable category.
- **P22-D3 (cache location + cap):** App-Group `UserDefaults` (`group.com.reojacob.myhome`), mirroring `DismissedMessageStore` — not a SwiftData model, because a SwiftData entity is exactly the synced, migrating state that ICON-03 forbids. Capped at 300 entries with least-recently-used eviction; a household pantry is tens of items, so the cap only ever bites on typo churn.
</locked_decisions>

Create `MyHomeApp/Features/Kitchen/PantryIconCache.swift`.

Declare `struct PantryIconCache: Sendable` with an injectable `let defaults: UserDefaults`, defaulting to `UserDefaults(suiteName: "group.com.reojacob.myhome") ?? .standard` exactly as `DismissedMessageStore` does. Injection is what lets the tests use an isolated suite instead of stamping on the real app group.

Storage: one key `pantry_icon_categories` holding `[String: String]` (normalised name → `PantryCategory.rawValue`), and one key `pantry_icon_recency` holding `[String]` ordered oldest-first. Two plain plist-safe values, no `Codable`, no archiver — same discipline as the existing stores.

API: `func category(forName: String?) -> PantryCategory?`, `func store(_ category: PantryCategory, forName: String?)`, `func removeAll()`, and `var count: Int` for tests. Every entry point normalises via `KitchenLogic.normalizedIconKey(forName:)` and no-ops on nil so a blank name can never occupy a slot.

`store` semantics: write the raw value into the map, remove any prior occurrence of the key from the recency array and append it to the end, then while the map exceeds `maxEntries` (a `static let maxEntries = 300`) drop the head of the recency array and its map entry. Write both keys back.

`category(forName:)` reads the map and returns `PantryCategory(rawValue:)` — an unrecognised raw value yields nil by construction, which is the 22-01 behaviour this relies on. Reading does NOT touch recency; keep reads free of writes so row rendering never writes to UserDefaults.

ICON-03 note in the file header: this cache is the ONLY place a classification is persisted. `PantryItem` gains no field, `SchemaV11` does not change, and no sync DTO mentions icons — so two phones may briefly disagree on a tile until each classifies locally, which is harmless and self-correcting.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-typed item name → model prompt | Free text crosses into a language-model prompt |
| classification result → device-local persistence | Model output is written to App-Group UserDefaults |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-03 | Tampering | `PantryIconPromptBuilder.buildPrompt` | mitigate | Name truncated to 80 chars and injected as the sole datum; output is a closed `@Generable` enum, so a prompt-injection attempt can at worst change which of 17 icons is drawn. |
| T-22-04 | Information disclosure | `FoundationModelsPantryIconClassifier` | mitigate | On-device `SystemLanguageModel` only — no `URLSession`, no network entitlement. Only the single item name enters the prompt; no quantity, no other row, no financial data. |
| T-22-05 | Tampering | `PantryIconCache` persisted raw values | mitigate | Reads go through `PantryCategory(rawValue:)`; an unrecognised or hand-edited value returns nil and falls back to the keyword table rather than crashing. |
| T-22-06 | Denial of service | UserDefaults growth | mitigate | 300-entry LRU cap enforced on every `store`. |
| T-22-SC | Tampering | dependencies | accept | Zero new packages; `FoundationModels` is a first-party Apple SDK already linked in Phase 16. |
</threat_model>
