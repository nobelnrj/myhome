## Spacing Scale

**Inherited from Phase 1 UI-SPEC — no changes.** Binding values:

| Token | Value | SwiftUI Expression | Usage |
|-------|-------|--------------------|-------|
| xs | 4pt | `.padding(4)` / `spacing: 4` | Icon gaps, inline label padding |
| sm | 8pt | `.padding(8)` / `spacing: 8` | Compact element spacing, row internal gaps |
| md | 16pt | `.padding(16)` / `spacing: 16` | Default section padding, form field spacing |
| lg | 24pt | `.padding(24)` / `spacing: 24` | Card internal padding, section breaks |
| xl | 32pt | `.padding(32)` / `spacing: 32` | Screen-level horizontal margins |
| 2xl | 48pt | `spacing: 48` | Major vertical section separators |
| 3xl | 64pt | `spacing: 64` | Full-bleed hero areas (not used in Phase 2) |

**Phase 2 additions:**

| Context | Value | Rationale |
|---------|-------|-----------|
| Progress bar height | 8pt | Compact but legible; matches iOS native `ProgressView` slot height. Render as a filled `RoundedRectangle(cornerRadius: 4)` overlay on a `secondarySystemBackground` track of the same height. |
| Category icon frame | 28pt × 28pt | SF Symbol in a circle/square frame on budget cards. Slightly larger than the 24pt picker list icon to give the card visual weight. |
| Budget card corner radius | 12pt | Matches Phase 1 `amountSection` card corner radius — visual consistency. |
| Month pager touch target | 44pt × 44pt minimum | Apple HIG minimum; `<` / `>` chevron buttons must meet this. |
| Tab bar | System-managed | Do not set custom heights; TabView respects system tab bar metrics. |

Exceptions:
- Minimum tappable touch target: 44pt × 44pt (inherited from Phase 1).
- DecimalKeypadView keys: 64pt × 64pt (reused for budget-amount entry — inherited from Phase 1).
- Navigation bar back button: system-managed (inherited from Phase 1).

| Role | SwiftUI Color | Light Hex (ref) | Dark Hex (ref) | Usage % | Usage |
|------|---------------|-----------------|----------------|---------|-------|

**Phase 2 semantic additions (D2-09 — budget progress color thresholds):**

These extend the Phase 1 semantic-color system. Color is NEVER the sole signal — each threshold is
paired with the ₹-remaining and % text values.

| Semantic Role | SwiftUI Color | Light Hex (ref) | Dark Hex (ref) | Threshold | Usage |
|---------------|---------------|-----------------|----------------|-----------|-------|
| Budget normal | `Color.accentColor` (`.systemIndigo`) | `#5856D6` | `#7B79E6` | `fractionUsed < 0.80` | Progress bar fill; matches Phase 1 accent |
| Budget warning | `Color(.systemOrange)` | `#FF9500` | `#FF9F0A` | `0.80 ≤ fractionUsed < 1.00` | Progress bar fill; amber signal |
| Budget over | `Color(.systemRed)` | `#FF3B30` | `#FF453A` | `fractionUsed ≥ 1.00` | Progress bar fill; reuses destructive token |

Implementation mapping (`BudgetColor` enum from RESEARCH.md):

```
BudgetColor.normal      → Color.accentColor
BudgetColor.warning     → Color(.systemOrange)
BudgetColor.overBudget  → Color(.systemRed)
```

Progress bar track (background): `.secondarySystemBackground` — always rendered at 8pt height,
`RoundedRectangle(cornerRadius: 4)`.

**Updated Accent reserved-for list (extends Phase 1):**
1. "Add Expense" primary CTA button (inherited Phase 1)
2. Amount field cursor and active ring (inherited Phase 1)
3. "Save Expense" button in NavigationStack toolbar (inherited Phase 1)
4. Keypad "confirm amount" submit action (inherited Phase 1)
5. Tappable date in expense row (reserved Phase 1)

### Screen 3 (New): Budgets Tab — BudgetsView

**Navigation:** Second tab, owns its own `NavigationStack`.
**Purpose:** Per-category budget progress for the selected month. Entry point for tap-through and management. Satisfies EXP-08 + EXP-09. Source: D2-10.

Layout contract:
- `NavigationStack` with `.navigationTitle("Budgets")` in `.inline` display mode.
- **Month pager header** (pinned below navigation bar, above the list):
  - `HStack` with:
    - `<` button (SF Symbol `chevron.left`, `.secondary` tint, 44pt touch target): previous month
    - Month + year label: "May 2026" — `.font(.title2)` semibold, centered, expands to fill
    - `>` button (SF Symbol `chevron.right`, `.secondary` tint, 44pt touch target): next month
  - The `>` button is disabled when `viewedMonth` equals the current calendar month (no future-month navigation).
  - Background: `.systemBackground`; subtle `.separator` divider below.
- **Category card list**: `List` with `.listStyle(.insetGrouped)`.
  - Sorted by `Category.sortOrder` (predefined 0–13 first, then custom categories by creation order).
  - Each card: `BudgetCategoryCard` (see Card spec below).
  - "Uncategorized" row at the bottom of the list: shows sum of uncategorized spend, no progress bar, no budget. Visible only when there is at least one uncategorized expense in the viewed month. Source: D2-08.
  - Toolbar trailing: `Button("Manage") { … }` — presents `ManageCategoriesView` as a `.sheet`. Label: "Manage" (text, not icon-only — provides accessible copy). Source: D2-11.

**BudgetCategoryCard layout:**
- `VStack(alignment: .leading, spacing: 8)` inside a List row, `padding(md = 16pt)`.
- Row 1: `HStack`
  - SF Symbol icon in a 28pt × 28pt frame, `.font(.body)`, `.secondary` foreground
  - Category name: `.font(.body)`, `.primary`, one line truncated
  - Spacer
  - Spent amount: `expense.amount.formattedINR()` — `.font(.subheadline)`, `.secondary` — shows month's total spend
  - "Edit" button (SF Symbol `pencil`, `.secondary`, 44pt touch target): presents `EditBudgetSheet`
- Row 2 (progress bar — only rendered when `monthlyBudget != nil`):
  - `GeometryReader` + `ZStack` — track (`.secondarySystemBackground`, 8pt height, `RoundedRectangle(cornerRadius: 4)`) behind fill (`BudgetColor`-matched color, capped at full width at 100%).
  - Fill width: `min(fractionUsed, 1.0) * availableWidth` — never overflows the track.
  - Animated: `.animation(.easeInOut(duration: 0.3), value: fractionUsed)`
- Row 3 (budget detail — only rendered when `monthlyBudget != nil`):
  - `HStack`
    - `"₹X remaining"` — `remaining.formattedINR()` — `.font(.subheadline)`, color matches `BudgetColor` token (normal → `.secondary`, warning → `.systemOrange`, over → `.systemRed`)
    - Spacer
    - `"X% used"` — `"\(Int(fractionUsed * 100))%"` — `.font(.subheadline)`, `.secondary`
- Row 2 / 3 (no budget — `monthlyBudget == nil`):
  - Single line: "No budget set" — `.font(.subheadline)`, `.tertiary` foreground
  - "Edit" pencil button still present so user can set a budget.
- Card background: `.secondarySystemBackground`, `RoundedRectangle(cornerRadius: 12)`, `shadow(color: .black.opacity(0.04), radius: 2, y: 1)`.

**Uncategorized row** (appended after category cards when present):
- Simplified row (no progress bar, no edit button):
  - Label: "Uncategorized" — `.font(.body)`, `.primary`
  - Value: total uncategorized spend for the month — `.font(.subheadline)`, `.secondary`
- Tapping opens `FilteredExpenseListView` with `category: nil` predicate (in-memory filter).

## Interaction Contracts

### TabView Navigation (D2-10)

- Tab state is preserved independently: navigating away from the Budgets tab and back returns the user to the same `viewedMonth` and scroll position.
- Deep-linking from expenses to a budget category drill-through is NOT required in Phase 2 (deferred).
- The `+` add expense button remains accessible from the Expenses tab toolbar (inherited).

### Month Paging (BudgetsView)

- Default state: current calendar month.
- `<` button: decrements `viewedMonth` by 1 month. Always enabled (no earliest-month limit in v1).
- `>` button: increments `viewedMonth` by 1 month. Disabled when `viewedMonth == currentMonth` (no future browsing).
- Changing `viewedMonth` triggers a re-fetch via the `BudgetsMonthView(start:end:)` sub-view reinit (RESEARCH.md Pattern 3 / Open Question 3 resolution). The transition is instant — no animation on the card list redraw (avoids visual thrash during data reload).
- Month label format: `"MMMM yyyy"` via `DateFormatter` — "May 2026". Source: extend `Date+Display.swift`.

### Category Picker Interaction (D2-12)

- Category row in Add/Edit sheet: single tap → sheet presentation. No double-tap, no long-press.
- Selecting a category: immediate row update + sheet dismissal (no secondary confirm).
- Clearing a category ("None" row or "Clear" toolbar button): immediate update + dismissal.
- The category picker never blocks the ≤4-tap add path (tapped only when user actively wants to categorize).

### Budget Progress Bar

- `fractionUsed` capped at 1.0 for bar width calculation (bar never overflows the track even if 200% over budget).
- Color transitions are immediate (no cross-fade) — value changes are discrete, not animated.
- Progress bar ALWAYS paired with `"₹X remaining"` and `"X% used"` text — color is never the sole signal (D2-09, accessibility).
- When `spent == 0` and `budget > 0`: bar is empty (0%), "₹{budget} remaining", "0% used". No special empty-bar state.
- When `monthlyBudget == nil`: no bar rendered. "No budget set" text instead (`.tertiary`).

### Category CRUD Interactions (D2-11, EXP-05)

- **Add:** Tap `+` in ManageCategoriesView toolbar → new row with empty TextField auto-focused. Tap "Done" on keyboard row or toolbar to confirm. Empty names are rejected (inline: "Category name cannot be empty.").
- **Rename:** Tap existing row → TextField appears pre-filled. Losing focus (tapping elsewhere) OR tapping "Done" saves the rename. Duplicate-name check runs on save.
- **Delete:** Swipe left → "Delete" action (`.systemRed`). Tapping "Delete" presents confirmation action sheet. On confirm: category is deleted, expense links are nullified (D2-04), associated budget is removed.

### Edit Budget Interactions

- DecimalKeypadView reused (same component, same key layout).
- "Save Budget" enabled only when amount > 0 (same rule as expense amount gate).
- "Remove Budget" confirmation prevents accidental budget deletion (action sheet pattern, same as Phase 1 delete confirmation).

## Accessibility Contracts

**Inherited from Phase 1:**
- All icon-only interactive elements have explicit `.accessibilityLabel`.
- Color is never the sole differentiator.
- Minimum touch target 44pt × 44pt.
- Dynamic Type on all text.

**Phase 2 additions:**
- Month pager buttons: `.accessibilityLabel("Previous month")` and `.accessibilityLabel("Next month")`.
- Budget progress bar: `.accessibilityLabel("\(category.name ?? "Category"), \(Int(fractionUsed * 100))% of budget used, ₹\(remaining.formattedINR()) remaining")`. Uses `accessibilityElement(children: .ignore)` on the `GeometryReader`/`RoundedRectangle` progress bar — the card row is the accessibility element.
- Category SF Symbol icons: `.accessibilityHidden(true)` (decorative; category name provides the label).
- "Edit" pencil button on budget card: `.accessibilityLabel("Edit budget for \(category.name ?? "category")")`.
- Tab bar: system-managed by `TabView` — accessible automatically.
- Progress bar color changes: each threshold must be paired with text (already required by D2-09). The accessible label on the progress bar row must also include the threshold state: e.g. "Warning: 85% of budget used" or "Over budget: 120% used" — prefix text only when `colorThreshold != .normal`.

|----------|-------------|-------------|

No third-party component registries. All UI is built from SwiftUI system primitives + SF Symbols.
All new visual components (`BudgetProgressView`, `BudgetCategoryCard`, `CategoryPickerView`,
`ManageCategoriesView`, `EditBudgetSheet`, `FilteredExpenseListView`) are built entirely from
SwiftUI primitives. No external packages introduced in Phase 2.
