| (no network / no auth / no IPC) | Local-only app |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-12 | Tampering | duplicate category insert (no `.unique`) | mitigate | Lookup-before-insert: case-insensitive `FetchDescriptor<Category>` name fetch before insert (P2-06); inline error, no insert on conflict |
| T-02-13 | Tampering | category delete cascade | mitigate | `.nullify` delete rule clears expense links (verified in CategoryCRUDTests); confirmationDialog prevents accidental delete; explicit `try context.save()` (CR-01) |
| T-02-14 | Tampering/DoS | relationship-contains predicate runtime failure | mitigate | Documented in-memory `.filter` fallback (RESEARCH OQ1/A3) at household data volumes — no crash, no perf risk |
| T-02-15 | Spoofing/Tampering | category name in lists/headers | mitigate | Plain `Text` rendering everywhere — never AttributedString(markdown:) |
| T-02-SC | Tampering | package installs | n/a | No installs — first-party SwiftUI/SwiftData only |

No HIGH threats. Local-only surface; name input validated (non-empty, unique-by-lookup), deletes confirmed and nullify-safe, fragile predicate has a tested fallback.
