## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local network → MCSession | Any device in Bonjour range can advertise/invite with the same service type; received frames are untrusted bytes |
| Info.plist local-network grant | User-consented capability; denial must degrade gracefully, not crash |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-01 | Information Disclosure | MCSession link | mitigate | `encryptionPreference: .required` — session refuses unencrypted links; asserted by grep gate + must_have |
| T-19-02 | Spoofing | invitation acceptance | accept | Unauthenticated DH encryption (no securityIdentity): a nearby device running this exact unpublished sideloaded app could connect. Two-phone household, app not distributed; connected peer name is surfaced in the Plan-03 UI so an unexpected peer is visible. Revisit with a pairing code if the app is ever distributed. |
| T-19-03 | Tampering | didReceive data | mitigate | Strict SyncEnvelope decode; malformed frames dropped with a `.failed` event, never crash; snapshot payloads additionally face Phase-18 SnapshotCodec version refusal before any store write |
| T-19-04 | Denial of Service | discovery/permission | mitigate | didNotStart* errors surface as `.failed` with Local Network settings hint; manual "Sync now" + Phase-18 AirDrop remain always-works fallbacks |
