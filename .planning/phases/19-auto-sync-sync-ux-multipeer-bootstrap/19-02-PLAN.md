    - Private state: `var isActive = false` (between start/stop),
      `var isMerging = false`, `var didSendThisConnection = false`,
      `var pendingPushTask: Task<Void, Never>?`, `var retryTask: Task<Void, Never>?`,
      `var retryDelay: TimeInterval` (current backoff, starts at retryBaseDelay).
    - `func start()`: guard !isActive; isActive = true; set `transport.onEvent =
      { [weak self] in self?.handle($0) }`; statusStore.status = .connecting;
      transport.start().
    - `func stop()`: isActive = false; cancel retryTask + pendingPushTask (nil them);
      transport.stop(); statusStore.status = .idle; statusStore.connectedPeerName = nil;
      didSendThisConnection = false. Idempotent.
    - `func handle(_ event: SyncTransportEvent)` — internal (not private) so tests
      can drive it directly:
      * `.connecting(peerName:)` → status .connecting.
      * `.connected(peerName:)` → connectedPeerName set; retryDelay reset to base;
        didSendThisConnection = false; then `pushLocalSnapshot()` (BOTH sides push
        on connect — merge is idempotent + LWW, so the double exchange converges in
        one round; this is also what makes bootstrap "just work" in Plan 04).
      * `.received(.snapshotRequest)` → `pushLocalSnapshot()` (unconditional reply —
        this is the "Sync now always works" path).
      * `.received(.snapshot(let data))` → status .syncing; set `isMerging = true`;
        `defer { isMerging = false }`; `let stats = try SnapshotImporter.mergeData(data,
        into: context)`; on success `statusStore.markSynced(stats: stats)` and, if
        `!didSendThisConnection`, `pushLocalSnapshot()` (covers the ordering where
        we received before our connect-push fired). On throw → status
        `.error(message:)` with the human-readable description; store untouched
        (engine guarantees no partial merge).
      * `.disconnected` → connectedPeerName = nil; if isActive → status .connecting
        and `scheduleRetry()`.
      * `.failed(message:)` → status .error(message); if isActive → `scheduleRetry()`.
    - `func pushLocalSnapshot()` — internal for tests: guard context set and
      `transport.isConnected`; status .syncing;
      `try transport.send(.snapshot(SnapshotExporter.exportData(context:, deviceName:)))`;
      on success `didSendThisConnection = true` + `statusStore.markSynced()`
      (a completed outbound send IS a sync from this phone's perspective); on throw
      → status .error.
    - `func localStoreDidChange()`: **CRITICAL echo-loop guard** — `guard !isMerging
      else { return }`. mergeData ends in `context.save()`, which fires didSave;
      without this guard two connected phones ping-pong full snapshots forever
      (merge → save → push → peer merges → peer saves → peer pushes → …).
      Then guard isActive && transport.isConnected; debounce: cancel pendingPushTask,
      `pendingPushTask = Task { try? await Task.sleep(for: .seconds(pushDebounce));
      guard !Task.isCancelled else { return }; self.pushLocalSnapshot() }`.
    - `func scheduleRetry()`: cancel prior retryTask; `retryTask = Task { try? await
      Task.sleep(for: .seconds(retryDelay)); guard !Task.isCancelled, self.isActive
      else { return }; self.retryDelay = min(self.retryDelay * 2, 30);
      self.transport.stop(); self.transport.start() }` — fresh MC objects per the
      19-01 transport contract; backoff 2s→4s→…→30s cap.
    - `func syncNow()` (manual fallback, SYNC-04): if transport.isConnected →
      `pushLocalSnapshot()` then `try? transport.send(.snapshotRequest)` (pull theirs
      too). Else → retryDelay reset to base; transport.stop(); transport.start();
      status .connecting (forces fresh discovery instead of silently doing nothing).

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-19-05 | Denial of Service | didSave → push loop | mitigate | isMerging suppression (merge-triggered saves never push) + debounce + bounded-exchange unit test; retry backoff capped at 30s |
| T-19-06 | Tampering | auto-merge of received snapshot | mitigate | All bytes go through Phase-18 SnapshotCodec strict decode + schema-version refusal BEFORE any store write; decode/merge failure → `.error` status, store untouched (tested with garbage bytes) |
