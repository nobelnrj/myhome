# Phase 13 — UI Design Contract: Design System Foundation

> Visual and interaction contract for the neumorphic design system infrastructure.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Platform:** Native iOS (Swift 6.2 / SwiftUI / SwiftData, iOS 17+ deployment target).
> All token values are translated from the canonical design handoff at

- Icon tile sizes: 26–40pt square, radius ≈ 28% of size (≈ `size * 0.28`)

## Typography

All sizes are expressed as SwiftUI Dynamic Type roles or `@ScaledMetric` values. No view file
may hardcode a numeric pixel/point size outside `DesignTokens.swift`.

Source: handoff README typography table (neuro section).

| Role | SwiftUI API | Base size | Weight | Tracking / Line Height | Notes |
|------|------------|-----------|--------|----------------------|-------|
| `largeTitle` | `.font(.largeTitle)` | 34pt | `.bold` (700) | tracking 0.37pt, lh 1.1 | Screen collapsing large title |
| `sectionHeader` | `@ScaledMetric` anchor 22pt | 22pt | `.bold` (700) | tracking −0.4pt | Card section titles |
| `cardTitle` | `@ScaledMetric` anchor 16pt | 16pt | `.semibold` (600) | tracking −0.3pt | Card and row primary titles |

| `subheadline` | `.font(.subheadline)` | 15pt | `.regular` (400) | — | Supporting row text |
| `caption` / `subtitle` | `.font(.caption)` | 13pt | `.regular` (400) | tracking −0.08pt | Section headers, footers, subtitles |
| `eyebrow` | `@ScaledMetric` anchor 11.5pt | 11.5pt | `.semibold` (600) | uppercase, tracking 1.2pt | "NET CASH FLOW", "JUNE 2026" labels |
| `tabLabel` | `@ScaledMetric` anchor 10pt | 10pt | `.medium` (500) active `.semibold` (600) | — | Tab bar labels |
| `heroMoney` | `@ScaledMetric` anchor 46pt | 46pt | `.ultraLight` (200) | tracking −2pt | Hero rupee readout (`RollingMoneyText`) |
| `statNumber` | `@ScaledMetric` anchor 21pt | 21pt | `.light` (300) | tracking −0.6pt | Stat tiles (Income/Spent sub-amounts) |

Key rules:
- `heroMoney` and `statNumber` use `fontDesign(.rounded)` to soften ultra-light numerals.
- All monetary numerals use `.monospacedDigit()` (tabular figures equivalent).
- Text in `fontWeightRange`: only two body-weight steps used per component group — never more than 3 weights in a single view.
- Dynamic Type scaling: `@ScaledMetric` anchored values scale with the user's preferred text size, capped at accessibility size xxxLarge where layout integrity requires (handled per-view via `minimumScaleFactor`).

## Shadow System

Source: `tokens.jsx` neuro branch. Translated to SwiftUI `.shadow()` and `ShadowStyle`.
Phase 13 defines these as named token constants so no view ever hardcodes shadow values.

### Raised Card (standard card/tile)

```
Light shadow (top-left): color white opacity 0.035, x -6, y -6, blur 14
Dark shadow (bottom-right): color black opacity 0.55, x 7, y 7, blur 18
```

SwiftUI implementation:
```swift
// Two .shadow() modifiers stacked, or a custom shader path
.shadow(color: .white.opacity(0.035), radius: 7, x: -6, y: -6)
.shadow(color: .black.opacity(0.55), radius: 9, x: 7, y: 7)
```
Named constant: `DesignTokens.shadowRaised`

### Floating Element (hero card, tab bar capsule, bottom sheet)

```
Light shadow: color white opacity 0.04, x -9, y -9, blur 22
Dark shadow: color black opacity 0.62, x 11, y 11, blur 28
```
Named constant: `DesignTokens.shadowFloat`

### Inner Rim (raised surface top-left highlight)

```
Inner light: inset, color white opacity 0.045, x 1, y 1, blur 1
Inner dark: inset, color black opacity 0.30, x -1, y -1, blur 1
```
SwiftUI: overlay `RoundedRectangle` stroke — see `NeuSurface` implementation note below.
Named constant: `DesignTokens.shadowRim`

### Recessed Well (tracks, inputs, progress bars)

```
Inner dark: inset, color black opacity 0.50, x 2, y 2, blur 5
Inner light: inset, color white opacity 0.035, x -2, y -2, blur 5
```
SwiftUI: overlay with inner `RoundedRectangle` + inset shadow approximation.
Named constant: `DesignTokens.shadowRecessed`

> SwiftUI note: SwiftUI's `.shadow()` modifier does not support inset shadows natively.
> `NeuSurface(.recessed)` implements the inset look via a CALayer shadow in a `UIViewRepresentable`
> wrapper, OR via an inverted-blend overlay — the planner must decide the implementation approach
> and capture it in the plan. The visual result must match the recessed spec above.

## NeuSurface ViewModifier Contract (DS-02)

`NeuSurface` is a SwiftUI `ViewModifier` that wraps any content in a neumorphic surface.
It replaces `CardStyle` entirely. **`CardStyle` must not be used in any new v1.2 code.**

### States

| Modifier call | Fill | Shadow | Inner rim | Use for |
|--------------|------|--------|-----------|---------|
| `.neuSurface(.raised)` | `surfaceRaised` | `shadowRaised` (dual outer) | `shadowRim` overlay | Default cards, list containers, row groups |
| `.neuSurface(.floating)` | `surfaceRaisedStrong` | `shadowFloat` (dual outer, deeper) | `shadowRim` overlay | Hero card, tab bar capsule body, modal sheets |
| `.neuSurface(.recessed)` | `fillRecessed3` | `shadowRecessed` (dual inset) | none | Progress tracks, text inputs, inset wells |

### Additional parameters

```swift
func neuSurface(
    _ state: NeuSurfaceState,               // .raised / .floating / .recessed
    radius: CGFloat = DesignTokens.radiusCard,
    padding: CGFloat? = 16                  // nil = caller handles padding
) -> some View
```

### Accessibility requirement (DS-06)

When used as an interactive surface (button, tappable card), the modifier adds a 0.5pt
`Color.white.opacity(0.025)` border stroke as an additional boundary affordance — this is the
`glassBorder` token, and it ensures WCAG 1.4.11 (3:1) non-text contrast is never dependent on
shadow depth alone. Verified via Xcode Accessibility Inspector: zero contrast warnings required
before merge.

## NeuTabBar Component Contract (DS-03)

Replaces SwiftUI `TabView` chrome in `RootView`. Implemented as a custom view, not a ViewModifier.

### Structural layout

```
Floating capsule, anchored to bottom of screen:
- Horizontal position: centered
- Bottom inset: 24pt from screen bottom (safe area aware — see below)
- Height: 62pt
- Corner radius: 34pt
- Body fill: surfaceRaisedStrong (#22222C)
- Shadow: shadowFloat (dual outer)
- Inner rim: shadowRim overlay
- Optional border: glassBorder 0.5pt stroke

Per tab button:
- Width: 58pt
- Layout: icon (24pt SF Symbol) above label (10pt)
- Gap icon→label: 3pt
- Padding top+bottom: 7pt each side

Active sliding highlight pill:
- Dimensions: 58pt wide × (62 − 12)pt = 50pt tall
- Corner radius: 26pt
- Fill: accentSoft (rgba(255,214,10,0.16))
- Border: Color(hex:"#FFD60A").opacity(0.35) at 0.5pt stroke
- Glow: .shadow(color: accent.opacity(0.22), radius: 8)
- Animation: spring-based position transition, 0.4s
```

### Tab definitions (5 tabs — indices stable, source: `app.jsx` TABS array)

| Index | id | Label | SF Symbol (inactive) | SF Symbol (active) |
|-------|----|-------|---------------------|---------------------|
| 0 | `home` | Home | `house` | `house.fill` |
| 1 | `expenses` | Activity | `creditcard` | `creditcard.fill` |
| 2 | `budgets` | Budgets | `chart.pie` | `chart.pie.fill` |
| 3 | `notes` | Notes | `note.text` | `note.text` |
| 4 | `settings` | Settings | `gear` | `gear` |

Active tab: icon in `accent` (#FFD60A) color, label in `accent`.
Inactive tab: icon in `label2` (rgba(220,223,238,0.56)), label in `label2`.

### Safe-area inset contract (DS-03)

`NeuTabBar` floats over content using `.overlay(alignment: .bottom)` or a `ZStack`. Screen
content views must apply `.safeAreaInset(edge: .bottom, spacing: 0) { Color.clear.frame(height: 100) }`
(matching `TABBAR_H = 100pt` from handoff) so the last content row is never hidden behind the bar.
This is the responsibility of each screen's scroll view, not the tab bar itself.

### Deep-link tab index stability

The `tab` state variable retains integer/string indices matching v1.1. The existing deep-link
mechanism (RootView `@State tab` + Binding) continues to work unchanged — `NeuTabBar` is a
drop-in visual replacement that reads from and writes to the same binding.

## RollingMoneyText Component Contract (DS-04)

Animated headline money readout using `.contentTransition(.numericText(countsDown: …))`.

### API

```swift
RollingMoneyText(
    amount: Decimal,
    currencyCode: String = "INR",
    locale: Locale = Locale(identifier: "en_IN"),
    font: Font = .system(size: 46, weight: .ultraLight, design: .rounded),
    color: Color = Color(hex: "#ECEDF4"),
    animationDuration: Double = 0.78    // ~780ms, source: motion.jsx duration default
)
```

### Behavior contract

| Condition | Behavior |
|-----------|----------|
| Value changes | Animate via `.contentTransition(.numericText())` — digits roll from old to new |
| Reduce Motion enabled (`accessibilityReduceMotion`) | Value snaps to target immediately — zero intermediate frames; `.animation(nil)` applied |
| View appears (`.onAppear`) | No animation on mount; display target value immediately |
| Animation curve | SwiftUI spring `.smooth(duration: 0.78)` — approximates handoff easeOutCubic |
| Number format | INR with lakh grouping: `FormatStyle.currency(code: "INR").locale(Locale(identifier: "en_IN"))` |
| Negative amounts | Displayed without sign (absolute value); caller applies `DesignTokens.negative` color |
| Signed amounts (net cash flow) | Caller passes `signed: true`; component prepends "+"/"-" in `positive`/`negative` color |

### Typography

Uses `heroMoney` scale: `@ScaledMetric(relativeTo: .largeTitle) var baseSize = 46`.
Letter-spacing: −2pt. Font design: `.rounded` for ultra-light weight legibility.
Stat variant (smaller readouts): base 21pt, weight `.light`, tracking −0.6pt.

### Reduce Motion implementation

```swift
@Environment(\.accessibilityReduceMotion) private var reduceMotion

var transition: ContentTransition {
    reduceMotion ? .identity : .numericText()
}
```

## Motion & Interaction Contract

All animations in Phase 13 components must:
1. Check `@Environment(\.accessibilityReduceMotion)` and skip/replace motion when `true`.
2. Use only SwiftUI-native animation APIs — no UIKit CAAnimation, no third-party libraries.

### Spring parameters (from handoff)

| Name | CSS original | SwiftUI equivalent |
|------|-------------|-------------------|
| `springBouncy` | `cubic-bezier(.34,1.32,.42,1)` | `.spring(response: 0.4, dampingFraction: 0.65)` |
| `springSoft` | `cubic-bezier(.32,.72,0,1)` | `.spring(response: 0.4, dampingFraction: 0.9)` |

### Per-component animation

| Component | Property | Duration / Spring | Reduce Motion |
|-----------|----------|-------------------|---------------|
| `NeuTabBar` active pill slide | `offset` | `springBouncy`, 0.4s | `.animation(nil)` → instant |
| `RollingMoneyText` digit roll | digit display | `.smooth(0.78)` | `.identity` transition |
| Sheet slide-up | `offset` | `springSoft`, 0.34s | Instant (no translate) |
| Push navigation | `offset` | `springSoft`, 0.4s | SwiftUI default (already reduced) |

### Press / tap feedback

Interactive surfaces (tappable cards, tab buttons, CTA buttons):
- `.buttonStyle(.plain)` with `.scaleEffect(isPressed ? 0.97 : 1.0)` and `.opacity(isPressed ? 0.88 : 1.0)`.
- No custom ripple effect — iOS native press feedback is sufficient for v1.2.

## Accessibility Contract (DS-06)

### Contrast requirements

- Interactive surface boundaries must meet WCAG 1.4.11 (3:1) non-text contrast.
- The 3.5%-white shadow (`rgba(255,255,255,0.035)`) is **never** the sole edge affordance.
- `NeuSurface` adds `glassBorder` (0.5pt, `Color.white.opacity(0.025)`) on interactive surfaces.
- Canary yellow `#FFD60A` against canvas `#1C1C23`: computed contrast ratio ≈ 10.5:1 (exceeds both 3:1 and 4.5:1). This is the primary active-state affordance.
- `accentOnYellow` (`#1A1404`) against `accent` (`#FFD60A`): computed contrast ratio ≈ 12.6:1 (use for text/icons on yellow backgrounds).
- Verification gate: Xcode Accessibility Inspector must show zero contrast warnings on `NeuSurface`, `NeuTabBar`, and their preview canvases before the phase exits.

### Dynamic Type

- No hardcoded numeric font sizes in any view file. All sizes go through `DesignTokens` constants (either `.font(.systemStyle)` or `@ScaledMetric`-anchored).
- `heroMoney` uses `@ScaledMetric(relativeTo: .largeTitle)` — scales with text size, does not clip.
- Layout must not break at accessibility sizes (xxxLarge) — test in previews at each size.

### Reduce Motion

- `RollingMoneyText`: value snaps to target (no intermediate frames) when `accessibilityReduceMotion` is `true`.
- Tab bar active pill: jumps to new position (no sliding animation).
- Sheet/push transitions: SwiftUI's built-in Reduce Motion handling applies automatically.
- Typewriter animation (Phase 16 AI card): defined here as a rule — instant appearance when `accessibilityReduceMotion` is `true`. Phase 16 must implement this.

### VoiceOver labels

- `RollingMoneyText`: provide `.accessibilityLabel("₹\(formatted amount)")` with the final formatted value (not mid-animation interpolated value).
- `NeuTabBar` tab buttons: `.accessibilityLabel("\(tab.label)")` + `.accessibilityHint("Tab \(index+1) of 5")`.
- `NeuSurface` on interactive elements: ensure `.accessibilityElement(children: .combine)` when wrapping tappable content.
