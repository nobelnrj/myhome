<objective>
STAB-02: Gmail sync crashes/stalls on large inboxes because `categoriesByName: [String: Category]` is captured ONCE before the message loop, and those `@Model` references can be invalidated across the `await fetch.getRawMessage(...)` suspension point. Additionally `ctx.save()` is called inside the per-message loop (N saves for N messages). This bug exists in BOTH sync paths: `syncAccount` (~421-545) and `legacySingleAccountSync` (~547-686).

Fix (D-04): replace the captured `[String: Category]` map with `[String: PersistentIdentifier]` (a Sendable value type, safe across suspension); after each `await`, re-resolve via `ctx.model(for: catID) as? Category`; move `ctx.save()` to a SINGLE call after the loop. Preserve per-message resilience (D-03): a bad message or a failed category re-fetch is skipped and processing continues.

Purpose: Eliminate the large-inbox crash/stall (success criterion #2).
Output: Both sync paths refactored, plus a regression test that fails pre-fix and passes post-fix (D-08).

OPEN QUESTION (researcher-flagged, A2): verify `ModelContext.model(for: PersistentIdentifier)` returns the live model (and `nil`/throws for a deleted ID) against Apple SwiftData docs BEFORE implementing. Fallback if the signature differs: `ctx.fetch(FetchDescriptor<Category>(predicate: #Predicate { $0.persistentModelID == catID }))`.
</objective>
