<objective>
STAB-01: Stop the live crash that occurs when a Note or NoteBlock is deleted (from any view) while the Notes calendar / DayAgendaView is open. The crash happens because `remindersOnDay` and `CalendarAggregator` access `@Model` stored properties (`reminderEnabled`, `reminderDate`, `blocks`, `text`, `isChecked`) on objects that SwiftData has already tombstoned, faulting on the deleted backing object.

Fix: apply the project's established `note.modelContext != nil` tombstone-guard idiom (from EditNoteView.swift:332) at every iteration site before touching any property, in both `CalendarView.remindersOnDay` and `CalendarAggregator`. Preserve the live-binding design (D-02) — do NOT snapshot. The empty-state fallback already exists; no new empty-state code is needed (D-01).

Purpose: Eliminate a reproducible production crash (success criterion #1).
Output: Guarded CalendarView + CalendarAggregator, plus regression tests that fail pre-fix and pass post-fix (D-08).
</objective>
