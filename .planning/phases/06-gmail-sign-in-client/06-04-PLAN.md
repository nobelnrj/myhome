    reverse-client-ID scheme (com.googleusercontent.apps.&lt;CLIENT_ID&gt;:/oauth2redirect) as the

      <name>Reverse-client-ID scheme (research recommendation)</name>

      <cons>Overrides the literal D6-04 value (still satisfies ING-01 intent: ASWebAuthenticationSession + PKCE)</cons>
    </option>

      <name>Keep myhome-oauth://callback (literal D6-04)</name>

      <cons>May be rejected at Cloud Console registration; risk if Google tightens enforcement (A1)</cons>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| app ↔ Google OAuth/token/Gmail endpoints | network; HTTPS only via URLSession |
| app ↔ system browser (ASWebAuthenticationSession) | credentials entered in OS-owned browser, never in app code |
| app ↔ Keychain | refresh token at rest |
| OS ↔ app via custom URL scheme | OAuth callback redirect |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04-KEYCHAIN | Information Disclosure | SystemKeychainStore | mitigate | kSecAttrAccessibleAfterFirstUnlockThisDeviceOnly (SEC-03); ThisDeviceOnly prevents backup/restore exfiltration |
| T-06-04-CALLBACK | Spoofing | custom URL scheme callback | mitigate | ASWebAuthenticationSession validates the returned URL matches the registered scheme before firing; `state` validated; reverse-client-ID preferred (checkpoint) |
| T-06-04-CODEINTERCEPT | Spoofing/Elevation | authorization code interception | mitigate | PKCE code_verifier (plans 02/03) makes an intercepted code useless |
| T-06-04-PII | Information Disclosure | connected-email userinfo fetch | accept | single email address shown only to the app owner; gmail.readonly scope; no broader PII |
| T-06-04-SC | Tampering | package installs | accept | zero external packages; all first-party Apple frameworks + raw Google REST |
</threat_model>
