<objective>
Build the pantry half of the kitchen surface (KTCH-01, KTCH-02): a neumorphic
pantry list (DesignTokens + NeuSurface, light + dark) with add/edit sheet,
per-row used/restocked steppers, per-item low-stock thresholds, and visual
low/out-of-stock flags — plus the navigation entry that makes the surface
reachable from day one.

Navigation decision (recorded here, consumed by 20-04/20-05): Kitchen is a
PUSHED surface from the Overview tab, not a 6th tab. The native TabView already
holds 5 tabs — the iOS maximum before a "More" spillover tab appears — and
Assets/Analytics set the exact precedent (Overview `navigationDestination` +
a DEBUG `-openAnalytics`-style launch hook). Existing deep-link tab indices
0–4 (`-startTab`) stay byte-identical.

Sync hygiene contract (from 18-04): every pantry delete goes through
`context.deleteSynced(_, kind: .pantryItem)` (the .pantryItem case shipped in
20-02) and every user edit of an existing item calls `.touch()` before save so
LWW resolves in favor of the newest human edit.

Purpose: after this plan a user can fully manage pantry stock and see at a
glance what is low/out — the state the 20-04 shopping list derives from.

Output: Features/Kitchen group live with pantry list + editor, Overview entry,
seeded sample pantry for screenshot self-verify, logic tests GREEN.
</objective>
