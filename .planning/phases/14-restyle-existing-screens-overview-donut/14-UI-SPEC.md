# Phase 14 — UI Design Contract

> Visual and interaction contract for the full neumorphic restyle of all 9 screen groups
> plus the Overview "Where it's going" spend donut. Generated by gsd-ui-researcher.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Custom — `DesignTokens.swift` (hand-translated from `design/design_handoff_myhome_neumorphic/src/tokens.jsx` neuro branch) |
| Preset | Not applicable (native SwiftUI + Swift Charts, no shadcn) |
| Component library | Apple SwiftUI + Swift Charts (`SectorMark`) |
| Icon library | SF Symbols (system icons via `systemImage:`) |
| Font | SF Pro (system font) — `.system(size:weight:design:)` only |

**Skin lock:** Neomorphism (opaque charcoal + dual soft shadow). NOT Liquid Glass / translucent / iOS 26
glassEffect. `DesignTokens.swift` and `NeuSurface.swift` are used AS-IS — no changes to the design
system components themselves. (Source: 14-CONTEXT.md Locked Decisions)

**Platform lock:** iOS 17+, dark-mode-only, Swift 6.2 / SwiftData, no schema changes.

## Spacing Scale

All values are the on-grid `DesignTokens` constants. Executors MUST reference the token name,
not a raw number, in every modifier.

| Token | CGFloat | Usage |
|-------|---------|-------|
| `DesignTokens.spacing4` | 4 | Icon-to-label micro-gap, bullet dot margins |
| `DesignTokens.spacing8` | 8 | Compact element spacing, chip padding |
| `DesignTokens.spacing16` | 16 | Default horizontal padding, card default padding |
| `DesignTokens.spacing24` | 24 | Section padding, intra-card vertical rhythm |
| `DesignTokens.spacing32` | 32 | Layout gaps, top-of-screen breathing room |
| `DesignTokens.spacing48` | 48 | Major section breaks (e.g. bottom of last card) |

**Structural exceptions (handoff-sourced, from `DesignTokens`):**

| Token | CGFloat | Usage |
|-------|---------|-------|
| `DesignTokens.spacing2` | 2 | Tab bar icon↔label micro-gap |
| `DesignTokens.spacing12` | 12 | Active tab pill vertical inset |
| `DesignTokens.spacing22` | 22 | Inter-card vertical gap (VStack between top-level cards) |

**Touch targets:** All tappable icon buttons must be minimum 44×44pt (add `.frame(minWidth: 44, minHeight: 44)` where the visible icon is smaller — SF Symbols at 20–22pt are typically 44pt-contained via `.contentShape(Rectangle())`).

SwiftUI font calls only — no hardcoded pixel literals outside `@ScaledMetric` patterns. Dynamic Type
compliance is mandatory per DS-06.

## Per-Screen Restyle Contract

### Global Rules (apply to ALL screens)

1. Every `ScrollView` / `List` / `NavigationStack` background: `DesignTokens.bgCanvas` via
   `.scrollContentBackground(.hidden).background(DesignTokens.bgCanvas)` on `List` and
   `.background(DesignTokens.bgCanvas)` on `ScrollView`.
2. Every `.cardStyle(...)` call site migrated to `.neuSurface(.raised)` (see CardStyle Migration Map).
3. Navigation title style: `.navigationTitle("...")` with `.navigationBarTitleDisplayMode(.large)`.
   Title foreground: `DesignTokens.label`.
4. Toolbar items / nav bar buttons: foreground `DesignTokens.accent`.
5. List row backgrounds: `.listRowBackground(DesignTokens.surfaceRaised)`.
6. `Divider()` / row separators: use `DesignTokens.separatorHairline` (white 5% opacity).
7. Toggle: `.tint(DesignTokens.accent)`.
8. Progress/slider tint: `DesignTokens.accent` (on-budget), `DesignTokens.negative` (over-budget).

### Donut — "Where it's going" (OVR-05 + OVR-06)

**Implementation:** New `SpendDonutCard.swift` composing the existing `DonutChart.swift`.

**Data rules:**
- Query: current calendar month, `expense.isConfirmedSelfTransfer == false` (exclude confirmed self-transfers)
- Segments: top-4 spend categories by total (desc) + "Others" roll-up (all remaining categories summed)
- Minimum segment count: if fewer than 2 categories have any spend, show a single ring with a neutral segment
- Zero-spend month: show empty state (see Copywriting)

**Donut geometry:**
- `DonutChart` size: 132×132pt
- `innerRatio`: 0.62 (existing default)
- `angularInset`: 1.0pt (existing default)
- Corner radius per segment: 3pt (existing default)
- Card padding: 18pt on all sides
- Donut + legend layout: `HStack(spacing: 18)` — donut left, legend right (flex)
- Card must have no `clipped()` modifier on the outer container so shadow is not cut off

**Center overlay (passed as `center:` closure to `DonutChart`):**
```
VStack(spacing: 2) {
    Text("SPENT")
        .font(.system(size: 10.5, weight: .semibold))
        .tracking(0.6)
        .foregroundStyle(DesignTokens.label2)
    RollingMoneyText(amount: totalSpend, color: DesignTokens.label)
        // override @ScaledMetric base to 21pt for this instance
}
```
`RollingMoneyText` cannot directly change base size (it uses an internal `@ScaledMetric`). Use a
separate `Text` with `.font(.system(size: 21, weight: .light, design: .rounded)).monospacedDigit()`
alongside a `.contentTransition(.numericText())` instead of `RollingMoneyText` for the center stat.
(This matches the handoff `fontSize: 21, fontWeight: 300` center pattern.)

**Legend rows (up to 5 — 4 categories + Others):**
- Bullet dot: 9×9pt, radius 3pt, background = segment color; glow: `.shadow(color: segColor.opacity(0.6), radius: 5)`
- Category name: 14pt `.regular` `DesignTokens.label`
- Amount: 14pt `.medium` `DesignTokens.label2` (right-aligned, tabular numerals)
- Row gap: 11pt vertical

**Segment colors:**
- Each of top-4: use `DesignTokens.cat*` for the matched category (see category palette table)
- "Others": `DesignTokens.catOther` (#94A3B8)

**Tap interaction (OVR-06):**
- The entire card is wrapped in a `Button` that triggers navigation per segment — OR each legend row is a `Button`
- Preferred implementation: `DonutChart` passes segment tap via a `selection: Binding<String?>` property that callers observe; legend rows are `Button` → set selection → caller navigates
- Navigation target: `ExpenseListView` pre-filtered to `categoryID = segment.categoryID`; use the existing filter mechanism (the `expenseSub` / category filter binding)
- "Others" tap: navigates to Activity with all non-top-4 categories selected (or no filter if that is not feasible — acceptable fallback per CONTEXT.md)
- Animation on tap: `.scaleEffect(isPressed ? 0.97 : 1.0, anchor: .center)` with `springBouncy`

**Section label above the card:** "Where it's going" at 22pt `.bold` `DesignTokens.label`, padding `0 16px 12px`.

**Empty state (zero spend this month):**
- Icon: `circle.dashed` at 40pt `DesignTokens.label3` centered in donut space
- Subtitle inside card: 14pt `DesignTokens.label2` "No spend this month"

## CardStyle Migration Map

## Accessibility

DS-06 compliance is maintained by construction from Phase 13. Phase 14 must NOT regress it.

**Rules to preserve:**

1. Every tappable card that uses `.neuSurface(.raised)` and is interactive: pass `isInteractive: true` so the 0.5pt `glassBorder` stroke is rendered — this satisfies WCAG 1.4.11 (3:1) without relying on shadow depth alone.
2. Canary yellow `#FFD60A` on charcoal `#1C1C23` = ~9.7:1 contrast ratio (exceeds WCAG AA text and AAA for large text). Use for all active/selected states.
3. `RollingMoneyText` already honors `accessibilityReduceMotion` — no changes needed.
4. Donut `DonutChart` is marked `.accessibilityHidden(true)` — the legend rows must carry accessible labels.
5. Category legend rows in `SpendDonutCard` require `.accessibilityLabel("\(category): ₹\(amount)")` on each legend row.
6. All navigation buttons must have `.accessibilityLabel(...)` if they are icon-only (chevrons, plus, bell, trash).
7. Dynamic Type: no hardcoded pixel literal sizes in any view file. Use `.font(.system(size: X, weight: Y))` only for sizes that are then embedded in `@ScaledMetric` patterns or that match a named TextStyle.
