<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AirDrop/Files → onOpenURL | Anyone within AirDrop range can send a .myhomesnap; the file is fully untrusted until decoded and user-confirmed |
| exported file at rest | The snapshot contains the household's complete financial data (incl. rawEmailBody) as JSON |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-11 | Spoofing | unsolicited snapshot import | mitigate | Import NEVER auto-merges: decode-then-confirm sheet shows source device + counts; merge requires an explicit tap. Refused/malformed files never reach the importer. |
| T-18-12 | Information Disclosure | snapshot file contents | accept | Two-phone household transfer; AirDrop is peer-to-peer encrypted in transit; file is user-controlled and transient (temp dir/Files). Document in SUMMARY: users should delete stray snapshot files; at-rest encryption deferred to Phase 19 if Multipeer persists payloads. |
| T-18-13 | Tampering | crafted .myhomesnap | mitigate | SnapshotCodec strict decode + version refusal (Plan 02, tested); store untouched on any decode failure — verified by UI error paths. |
| T-18-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs — UniformTypeIdentifiers/UIKit/SwiftUI are first-party. |
</threat_model>
