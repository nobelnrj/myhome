<objective>
Define the transport-agnostic snapshot document layer (SYNC-02): a pure Codable
`SyncSnapshot` (schema-version-stamped, Decimal-as-string, deterministic canonical
encoding) plus the pure LWW conflict policy with a deterministic, convergent
tiebreak. Everything in this plan is value types testable with zero SwiftData
containers — this is the layer Phase 19's MultipeerConnectivity transport reuses
byte-for-byte.

Exported contracts (Phases 19/20 depend on these exact names):
`SyncSnapshot`, `SyncSnapshot.currentSchemaVersion`, per-entity DTOs
(`CategoryDTO` … `RoutineCompletionDTO`, `DeletionDTO`), `SyncEntityKind`,
`SyncDecimal`, `SyncError`, `SnapshotCodec.encode/decode`,
`SyncMergePolicy.remoteWins/tombstoneWins`.

Purpose: the codec and policy are where money corruption (JSON Double) and
non-deterministic merges would be born — locking them down as pure, byte-asserted
units before the engine exists kills both classes of bug at the source.

Output: 2 app files + 2 GREEN pure-unit test files, all pbxproj-registered under a
new Sync group.
</objective>

    - `static func remoteWins(localUpdatedAt: Date, localCanonical: Data,
      remoteUpdatedAt: Date, remoteCanonical: Data) -> Bool` — last-writer-wins:
      `remoteUpdatedAt > localUpdatedAt` → true; `<` → false; EXACT tie → compare
      canonical bytes lexicographically (`remoteCanonical.lexicographicallyPrecedes`
      logic): the lexicographically GREATER canonical encoding wins; identical
      bytes → false (no-op — this is what makes re-importing your own snapshot
      idempotent). Document WHY the tiebreak must not be "local wins"/"remote wins":
      both phones must pick the SAME winner from opposite vantage points or ties
      never converge.
    - `static func tombstoneWins(deletedAt: Date, recordUpdatedAt: Date) -> Bool`
      — `deletedAt >= recordUpdatedAt`. A delete beats the record state it deleted
      (and an exact tie: deletion wins — SYNC success criterion 2, no resurrection);
      a record EDITED strictly after the deletion survives (edit-after-delete
      resurrection is intentional LWW behavior, document it).
