<objective>
Wire the live write paths into the sync model: route every delete of a syncable
record through a tombstone-writing `deleteSynced` helper (SYNC-01's DeletionLog
becomes real, not just a table), and stamp `updatedAt` via `touch()` at user-edit
save paths so the Plan-03 LWW engine has an honest clock to resolve with (SYNC-02).

Exported contract: `ModelContext.deleteSynced<T: PersistentModel & SyncStamped>(_ model: T, kind: SyncEntityKind)`
— Phase 19/20 code deletes syncable records ONLY through this.

Purpose: without this plan the engine is correct but starved — deletes would
silently resurrect (no tombstones written) and edits would lose LWW to stale
timestamps. This is the difference between a tested engine and a truthful one.

Output: DeletionTracker + all 13 delete call sites converted + edit paths stamped +
behavior tests GREEN.
</objective>
