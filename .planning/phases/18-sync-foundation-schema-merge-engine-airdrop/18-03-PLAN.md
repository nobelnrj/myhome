  <action>
    Create `MyHomeApp/Sync/SnapshotImporter.swift`:

    `struct MergeStats: Equatable, Sendable { var inserted = 0; var updated = 0;
    var deleted = 0; var skipped = 0; var adopted = 0 }`

    `@MainActor enum SnapshotImporter` with
    `static func mergeData(_ data: Data, into context: ModelContext) throws -> MergeStats`
    (= decode via SnapshotCodec — version refusal happens there, BEFORE any store
    mutation — then merge) and
    `static func merge(_ snapshot: SyncSnapshot, into context: ModelContext) throws -> MergeStats`
    implementing, in this exact order:

    1. **Tombstone union**: fetch local DeletionLog, index by entitySyncID. For each
       remote DeletionDTO: absent locally → insert DeletionLog row; present → keep
       the max deletedAt. Result: the unified tombstone set (tombstones always
       propagate — that is how deletes reach the other phone).
    2. **Apply tombstones to local rows**: for every unified tombstone, if a local
       row of that entity kind has that syncID and
       `SyncMergePolicy.tombstoneWins(deletedAt:recordUpdatedAt:)` → `context.delete`
       it (count in stats.deleted). Deleting a tombstoned Note also drops its blocks
       via the existing cascade — correct, blocks were tombstoned by the tracker
       (Plan 04) anyway.
    3. **Identity adoption (deterministic, prevents day-one duplicates)**: for each
       remote DTO with NO local syncID match, look for a local row that is the same
       logical record created independently on both phones:
       - Category: equal trimmed, case-insensitive `name` (both phones seed the same
         14 categories at first launch — without this, first sync doubles every
         category).
       - Expense: equal non-nil `(sourceAccount, gmailMessageID)` pair (both phones
         ingest the SAME Gmail account independently — same bank mail, two syncIDs).
       On a match, pair them and set the LOCAL row's syncID to
       `min(local.syncID.uuidString, remote.syncID.uuidString)` (as UUID). The min
       rule is vantage-independent, so both phones converge to the same identity
       after one exchange in each direction. Count in stats.adopted, then fall
       through to LWW as a normal matched pair. No other entity types get adoption
       (their duplicates are legitimate distinct records).
    4. **Upsert pass 1 (scalars only)**: per entity type, fetch ALL local rows ONCE
       and index by syncID (fetch-then-upsert — never insert before checking).
       For each remote DTO: skip entirely if a unified tombstone wins over
       `dto.updatedAt` (stats.skipped — this is the no-resurrection path for an
       OLDER remote snapshot, SYNC success criterion 2). No local match → construct
       the model via its designated init, then assign EVERY stored property from
       the DTO (id, syncID, updatedAt, createdAt included; Decimal via
       `SyncDecimal.decimal(from:)`, refusing the row with malformedSnapshot on nil),
       `context.insert` (stats.inserted). Local match →
       `SyncMergePolicy.remoteWins(localUpdatedAt: local.updatedAt,
       localCanonical: canonical(exporterDTO(local)), remoteUpdatedAt:
       dto.updatedAt, remoteCanonical: canonical(dto))`; true → overwrite all
       scalar fields incl. updatedAt (stats.updated); false → leave the local row
       and its relationships untouched (stats.skipped).
    5. **Wiring pass 2 (create-then-link)**: after ALL entities exist, resolve
       relationships ONLY for rows inserted/updated in pass 1:
       `expense.categories = dto.categorySyncIDs.compactMap(categoryBySyncID[...])`;
       `block.note = noteBySyncID[dto.noteSyncID]`. A NoteBlock whose noteSyncID is
       tombstoned or unresolvable is NOT inserted (skip in pass 1 by pre-checking —
       no orphan blocks). LWW losers keep their local relationships.
    6. Single `try context.save()` at the end; any thrown error leaves the store
       unsaved (no partial merge committed). Return MergeStats.

    Keep every conflict DECISION in SyncMergePolicy (pure); this file only fetches,
    indexes, applies, and counts — the split is what keeps the engine unit-testable
    and transport-agnostic for Phase 19.
  </action>
