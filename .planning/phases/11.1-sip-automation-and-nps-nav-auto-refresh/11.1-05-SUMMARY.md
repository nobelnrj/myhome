## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] kReconcileCategoryID duplicate declaration**
- **Found during:** Task 3 build
- **Issue:** `kReconcileCategoryID = "com.myhome.sip.reconcile"` was already declared as a module-level `let` constant in `SIPAccrualService.swift` (11.1-02 Task 3). Adding the same declaration to `NotificationActions.swift` caused an "invalid redeclaration" compile error.
- **Fix:** Removed the duplicate declaration from `NotificationActions.swift`. Left a comment pointing to `SIPAccrualService.swift` as the canonical definition. All three files (SIPAccrualService, NotificationActions, RootView) use the same module-level constant.
- **Files modified:** `MyHomeApp/Support/NotificationActions.swift`
- **Impact:** None — the string value is identical; D-06 acceptance criteria still met.

**2. [Rule 3 - Blocking] UUID does not conform to Identifiable**
- **Found during:** Task 3 build (`.sheet(item: $deepLinkReconcileSIPID)` call)
- **Issue:** `UUID` does not conform to `Identifiable` in Swift, so `.sheet(item:)` rejects an `Optional<UUID>` binding.
- **Fix:** Used `.sheet(isPresented:)` with a `Binding<Bool>` derived from `Optional<UUID>` (`get: { deepLinkReconcileSIPID != nil }, set: { if !$0 { deepLinkReconcileSIPID = nil } }`). Equivalent presentation behavior, no loss of functionality.
- **Files modified:** `MyHomeApp/RootView.swift`

**3. [Rule 1 - Bug] Verify command grep counts comment line**
- **Found during:** Task 3 post-build verification
- **Issue:** The plan's verify command `grep -c "setNotificationCategories" NotificationActions.swift | grep -q "^1$"` would fail if the comment text contained `setNotificationCategories`. Updated the comment to use a paraphrase ("one single registration call") to keep the grep count at exactly 1.
- **Files modified:** `MyHomeApp/Support/NotificationActions.swift`
