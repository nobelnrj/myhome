# Phase 11.1 Plan 04: App Lifecycle Wiring + SIPSetupView Summary

Wired NPSNavService and SIPAccrualService into the app foreground chain, added NPS scheme picker and SIP entry to EditAssetView, and delivered the full SIPSetupView create/edit form with validated amount/day/allocation and amount-change history.

## What Was Built

**Task 1 — RootView lifecycle wiring:**
- `@State private var npsNavService = NPSNavService()` and `@State private var sipAccrualService = SIPAccrualService()` declared after existing services
- Both injected with `modelContext` in `onAppear`
- Foreground chain order: `routineResetService.resetIfNeeded()` → `amfiNavService.refreshIfNeeded()` → `npsNavService.refreshIfNeeded()` → `sipAccrualService.accrueIfNeeded()` → `netWorthSnapshotService.upsertIfNeeded()` (D-02: sipAccrual strictly before netWorth so same-cycle snapshot captures just-accrued units)
- `.environment(npsNavService)` added (SIPAccrualService does not need environment injection)

**Task 2 — EditAssetView additions:**
- `@Environment(NPSNavService.self) private var npsNavService` added (mirrors AMFINavService pattern)
- `@State private var npsSchemeCode: String? = nil` added
- NPS Scheme Section (conditional on `assetClassRaw == "nps"`): NavigationLink → NPSSchemePickerView bound to `$npsSchemeCode`, plain Text display (T-11-10)
- SIP Section (edit mode only, `asset != nil`): NavigationLink "Configure SIP" → SIPSetupView
- `onAppear`: `npsSchemeCode = a.npsSchemeCode`
- `saveAsset`: `target.npsSchemeCode = assetClassRaw == "nps" ? npsSchemeCode : nil` (clears on class switch)

**Task 3 — SIPSetupView full implementation:**
- `struct SIPSetupView: View` with `var asset: Asset`, `@Environment(\.modelContext)`, `@Environment(\.dismiss)`
- Form fields: amount (Decimal TextField), dayOfMonth (Stepper 1...31), startDate (DatePicker), isActive (Toggle)
- NPS section (conditional): E/C/G % TextFields, live sum hint with red/secondary color, strategy holding Pickers (all NPS assets via @Query)
- `static func validate(...)` — pure static validator for unit testing (T-114-01)
- `static func nextInstallmentDate(dayOfMonth:after:calendar:)` — pure static func, IST Gregorian, clamps dayOfMonth to month-end (Feb 28/29, etc.)
- `saveSIP()`: create → insert SIP + initial SIPAmountChange(effectiveFrom: startDate); edit → update SIP + append new SIPAmountChange(effectiveFrom: nextInstallmentDate) only when amount changed (D-07)
- `SIPSetupValidationTests.swift` registered in test target with 4 pbxproj edits; 15 tests all pass
