### 1. NPS NAV Live Refresh

**Test:** On iPhone 17 sim, foreground the app. Go to Assets list — find a holding with an npsSchemeCode linked. Observe currentNAV and navAsOfDate before and after.
**Expected:** After foreground, Asset.currentNAV reflects a value from npsnav.in; navAsOfDate updates to today IST. Assets without npsSchemeCode are unaffected. If the network is unavailable, the prior cached NAV is retained (no crash).
**Why human:** NPSNavService.refreshIfNeeded() fires against a live npsnav.in endpoint inside an async Task — cannot be verified offline.

### 2. NPS Scheme Picker and SIP Allocation Validation Flow

**Test:** Edit an NPS holding. Tap "NPS Scheme" — NPSSchemePickerView should show the fetched scheme list (or empty state + Fetch Now if not yet fetched). Select a scheme. Tap "Configure SIP" — SIPSetupView should open. Enter NPS allocations summing to ≠ 100 and verify Save is disabled. Change allocations to sum to 100 and verify Save is enabled.
**Expected:** Picker shows schemes searchable by name and code; selection persists back to EditAssetView. SIPSetupView live hint shows red "must total 100%"; Save disabled until allocation sums to exactly 100.
**Why human:** Multi-sheet navigation flow with live @Observable binding propagation — requires interactive UI.

### 3. SIP Accrual End-to-End

**Test:** Create a SIP on an MF holding (with a valid amfiSchemeCode) with a startDate in the past and an installment date that has already elapsed. Force-foreground the app or wait for scenePhase .active. Navigate to the holding's Contributions list.
**Expected:** ContributionLogView shows at least one row: date matches the elapsed installment date, amount matches SIP amount, navUsed is a real historical NAV, unitsAdded is positive (amount/navUsed rounded down to 4dp), isEstimate badge shows "Estimate". Asset.units has increased by unitsAdded.
**Why human:** Requires a live mfapi.in/npsnav.in historical NAV fetch with real scheme codes and elapsed dates.

### 4. Reconcile Units Flow

**Test:** With a holding that has Contribution rows, tap "Reconcile Units" from AssetDetailView. Enter an authoritative units total (e.g. "150.5000"). Tap "Confirm Units". Then open the Contributions list.
**Expected:** Asset detail view shows units = 150.5000. ContributionLogView still shows all prior Contribution rows unchanged (isEstimate badges still showing "Estimate" — not "Reconciled"). No rows were deleted.
**Why human:** Full AssetDetailView → ReconcileView → ContributionLogView round-trip requires live SwiftData state.

### 5. Reconcile Notification Deep-Link

**Test:** Set up a SIP that has accrued at least one Contribution. The SIPAccrualService will have called scheduleReconcileReminder(for:), scheduling a UNCalendarNotificationTrigger. Either wait for the trigger date or use `xcrun simctl push` with a synthetic notification payload containing `categoryIdentifier = "com.myhome.sip.reconcile"` and `userInfo["sipID"] = <SIP.id.uuidString>`. **Expected:** ReconcileView sheet opens for the correct holding. Cancel dismisses cleanly. The note reminder category remains unaffected (existing note reminders still work).
