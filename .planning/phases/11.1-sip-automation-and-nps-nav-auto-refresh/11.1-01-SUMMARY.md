# Summary: 11.1-01 — SchemaV8 Foundation

## What was built

The SchemaV8 foundation every other Phase 11.1 plan builds on:

- **SchemaV8.swift** — versioned schema (v8.0.0): copies V7's 7 models verbatim,
  adds `Asset.npsSchemeCode`, and introduces three new `@Model`s — `SIP`,
  `SIPAmountChange`, `Contribution` (all CloudKit-ready: defaulted/optional
  properties, no `@Attribute(.unique)`, `Decimal` money, UUID keys, bare-UUID
  back-refs instead of `@Relationship`).
- **MigrationPlan.swift** — appends `SchemaV8.self` to `schemas` and a `v7ToV8`
  additive `.custom(willMigrate: nil, didMigrate: nil)` stage (FB13812722 workaround
  preserved).
- **Atomic typealias flip** — all 10 model typealiases moved to SchemaV8 in one
  commit (STAB-08 discipline), plus 3 new typealias files (SIP/SIPAmountChange/Contribution).
- **pbxproj registration** — all 14 new files this phase introduces (4 app models +
  4 test files + 6 downstream service/view files) registered with the 4-edit rule so
  no downstream plan hits a silent compile-omission.
- **Wave-0 test scaffolds** — `SchemaV8MigrationTests` (BLOCKING, full V7→V8 round-trip),
  plus compiling NPSNavService/SIPAccrual/Reconcile scaffolds for downstream plans,
  with offline npsnav.in fixtures committed.

## Deviations from plan

These were not in the plan's task list but were required to get the schema bump to
actually run (the plan's task-2 verification was a pbxproj `grep`, not a build, so it
did not catch these):

1. **Container-schema declarations left at V7 (STAB-08 class crash).** The typealias
   flip + `AppMigrationPlan` bump to V8 left `ModelContainer.appContainer()` and every
   migration test's migrate-container at `Schema(versionedSchema: SchemaV7.self)`.
   SwiftData asserts the container schema must equal the migration plan's final version,
   so the app and all migration tests crashed at launch (`EXC_BREAKPOINT` /
   `_assertionFailure`). Fixed: bumped the production container + the migrate containers
   in MigrationTests / SchemaV6MigrationTests / SchemaV7MigrationTests / NoteModelTests
   to V8 (seed containers untouched).
2. **`@Model` initializers missing.** SIP/SIPAmountChange/Contribution had no `init`,
   which `@Model` requires. Added initializers with all params defaulted (matching the
   stored-property defaults) so the existing `SIP()`/`Contribution()` test calls compile.
3. **Pre-registered-but-nonexistent files broke the build.** The plan registered all 16
   phase files in pbxproj up-front (so downstream plans add content, not pbxproj refs).
   The 6 files created by plans 02–05 didn't exist yet → "Build input files cannot be
   found". Added compiling stubs for NPSNavService, SIPAccrualService, NPSSchemePickerView,
   SIPSetupView, ContributionLogView, ReconcileView; each downstream plan overwrites its
   stub with the real implementation.
