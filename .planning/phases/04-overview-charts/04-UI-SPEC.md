# Phase 4 — UI Design Contract

> Visual and interaction contract for Phase 4: Overview & Charts.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Platform:** Native iOS (SwiftUI). No web framework. shadcn gate not applicable.
> This contract EXTENDS Phases 1–3 (01-UI-SPEC.md, 02-UI-SPEC.md, 03-UI-SPEC.md).
> All prior tokens remain in force. Only Phase 4 additions and overrides are declared here.

## Spacing Scale

**Inherited from Phase 1 UI-SPEC — no changes to the core scale.** Binding values:

| Token | Value | SwiftUI Expression | Usage |
|-------|-------|--------------------|-------|
| xs | 4pt | `.padding(4)` / `spacing: 4` | Icon gaps, inline label padding |
| sm | 8pt | `.padding(8)` / `spacing: 8` | Compact element spacing, row internal gaps |
| md | 16pt | `.padding(16)` / `spacing: 16` | Card internal padding, section padding |
| lg | 24pt | `.padding(24)` / `spacing: 24` | Gap between dashboard cards |
| xl | 32pt | `.padding(32)` / `spacing: 32` | Screen-level horizontal margins |
| 2xl | 48pt | `spacing: 48` | Major section breaks (reserved) |
| 3xl | 64pt | `spacing: 64` | Full-bleed hero areas (not used in Phase 4) |

**Phase 4 additions:**

| Context | Value | Rationale |
|---------|-------|-----------|
| Dashboard card corner radius | 12pt | Matches `BudgetCategoryCard` from Phase 2 — visual consistency. |
| Dashboard card internal padding | 16pt (md) | Matches `BudgetCategoryCard.padding(16)` — visual consistency. |
| Card-to-card vertical gap | 16pt (md) | Tight enough for a dashboard feel without crowding. |
| Chart container height (spend-by-category) | 220pt | Fixed frame height to prevent chart from collapsing. Enough rows for up to 14 categories. |
| Chart container height (spend-over-time) | 200pt | Line chart needs vertical breathing room for shape clarity. |
| Quick-add `+` button touch target | 44pt × 44pt minimum | Apple HIG; placed in navigation toolbar. |
| Segmented control for chart ranges | System-managed | `Picker` with `.pickerStyle(.segmented)` — do not set a custom height. |
| Top-3 category row height | 44pt minimum | Apple HIG tap target; each row is tappable (no action v1, but future-safe). |
| Aggregate bar height | 16pt | Twice the Phase 2 progress bar (8pt) for visual prominence as the hero metric. |
| Aggregate bar track + fill corner radius | 8pt | Scaled up from Phase 2's 4pt to match the larger bar height. |

Exceptions (inherited from Phase 1):
- Minimum tappable touch target: 44pt × 44pt.
- Navigation bar back button: system-managed.

| Role | SwiftUI Style | Approx Size | Weight | Line Height | Usage |

| Context | Role | Notes |
|---------|------|-------|

| "of ₹X budget" label beneath the hero amount | Label (`.subheadline`) | `.secondary` foreground; qualifies the bar |

| Top-3 category row: rank indicator (#1, #2, #3) | Label (`.subheadline`) | `.secondary`; left of SF Symbol icon |
| Pinned note title | Body (`.body`) | One line, truncated with `…` |

| Chart axis labels (X and Y) | Label (`.subheadline` or `.caption`) | Use `.caption` (≈12pt) for axis values to avoid overlap at small widths; `.chart​XAxis` / `.chartYAxis` modifiers |
| Chart Y-axis ₹ labels | Label (`.caption`) | Use abbreviated form for readability: "₹0", "₹5k", "₹10k", "₹1L" (see Axis Formatting below) |

| Range segmented control labels ("Week", "Month", "Year") | System-managed | `Picker` with `.pickerStyle(.segmented)`; do not override font |

## Tab Bar Reorder (D4-01)

Phase 4 reorders the TabView. The executor MUST update `RootView.swift`:

| Tag | Label | SF Symbol | Notes |
|-----|-------|-----------|-------|
| 0 | "Home" | `house.fill` (active) / `house` (inactive) | NEW — Overview tab; default `selectedTab = 0` on app launch |
| 1 | "Expenses" | `list.bullet` | Shifted from tag 0 |
| 2 | "Budgets" | `chart.bar` | Shifted from tag 1 |
| 3 | "Notes" | `note.text` | Shifted from tag 2 — **CRITICAL: update `selectedTab = 2` deep-link constant in `RootView.swift` to `selectedTab = 3`** |

**Tab icon rationale (D4-08 discretion):** `house` / "Home" is chosen over `square.grid.2x2` / "Overview" because (a) "house" is the most universally understood "home" metaphor in iOS apps, (b) the app is literally called "My Home", and (c) `house.fill` / `house` is a standard SF Symbol pair for selected/unselected tab states. The `chart.bar.xaxis` alternative was considered but conflicts with the existing Budgets tab icon.

**`house` vs `house.fill` usage:** Use `.tabItem { Label("Home", systemImage: "house") }` — iOS automatically renders the filled variant for the selected state when using `Label` in `.tabItem`.

### Screen: Overview Tab — OverviewView

**Navigation:** Tab 0 ("Home"), default launch tab. Owns its own `NavigationStack`.
**Purpose:** Dashboard showing current-month household financial health at a glance. Satisfies OVR-01, OVR-02, OVR-03, OVR-04, EXP-10, EXP-11. Source: D4-03.

**Root layout:**
- `NavigationStack` with `.navigationTitle("Home")` in `.inline` display mode.
- Root body: `ScrollView(.vertical)` containing a `LazyVStack(spacing: 16)` of cards.
  - Use `LazyVStack` (not `VStack`) so charts below the fold are rendered lazily — avoids jank on scroll.
  - `LazyVStack(alignment: .leading, spacing: 16)` with `.padding(.horizontal, 16)` and `.padding(.top, 8)`.
- Navigation bar trailing: quick-add `+` button (SF Symbol `plus`; accessible label "Add Expense") — presents the existing `AddExpenseView` as `.sheet`. Source: D4-06, OVR-04.

**Card order (top → bottom, locked per D4-03):**
1. Spend vs. Budget card (OVR-01)
2. Top 3 Categories card (OVR-02)
3. Pinned Note card (OVR-03)
4. Spend by Category chart card (EXP-10)
5. Spend Over Time chart card (EXP-11)

Each card uses the standard card shell (see "Standard Card Shell" below).

All dashboard cards share:

#### Card 1: Spend vs. Budget (OVR-01)

**Data:** Current-month total spend vs. sum of all per-category `monthlyBudget` values. Source: D4-02, `BudgetCalculator`.

**Layout (VStack, alignment: .leading, spacing: 12):**

Row A — Card title:
- Text "This Month" — `.font(.title2)`, semibold

Row B — Hero spend amount:
- Text `totalSpend.formattedINR()` — `.font(.largeTitle)`, bold, `.primary`
- Text `"of \(totalBudget.formattedINR())"` — `.font(.subheadline)`, `.secondary`, same line (HStack) or directly below

Row C — Aggregate progress bar:
- `GeometryReader` + `ZStack(alignment: .leading)`:
  - Track: `RoundedRectangle(cornerRadius: 8).fill(Color(.secondarySystemBackground))`, height 16pt
  - Fill: `RoundedRectangle(cornerRadius: 8).fill(barFillColor)`, width `min(fractionUsed, 1.0) * geo.size.width`, height 16pt
  - `.animation(.easeInOut(duration: 0.3), value: fractionUsed)`
  - `barFillColor` MUST use `BudgetColor` mapping (D4-02): normal → `.accentColor`, warning → `.systemOrange`, over → `.systemRed`
- `.frame(height: 16)`
- `.accessibilityElement(children: .ignore)` on the bar geometry

Row D — ₹ remaining / over-budget label + % used (HStack):
- Left: `"₹X remaining"` or `"₹X over budget"` — `.font(.subheadline)`, color per `BudgetColor` (normal → `.secondary`, warning → `.systemOrange`, over → `.systemRed`)
- Right: `"X% used"` or `"100%+"` — `.font(.subheadline)`, `.secondary`
- Same logic as `BudgetProgressView` in Phase 2

**Empty state — no budget set (D4-07):**
- Row C and Row D are replaced by a single inline prompt:
  - Text "Set a budget to track your spending." — `.font(.subheadline)`, `.secondary`
  - A `Button("Set a budget") { … }` that navigates to the Budgets tab (switches `selectedTab = 2`) — `.font(.subheadline)`, `.accentColor` tint. No separate sheet.
- Row B still shows the current month's total spend (even without a budget, spend is real data)

**Empty state — no spend, no budget:**
- Row B shows "₹0.00" in the Display role
- Inline prompt: "No spend yet this month." — `.font(.subheadline)`, `.secondary`

**Accessibility:**
- `.accessibilityElement(children: .combine)` on the card `VStack` so VoiceOver reads as one unit
- Card accessible label: `"\(totalSpend.formattedINR()) spent this month, \(Int(fractionUsed * 100))% of \(totalBudget.formattedINR()) budget"` — or "No budget set" when applicable
- Bar fill: `.accessibilityElement(children: .ignore)` (geometry-level suppression; card label carries the value)

**Layout (VStack, alignment: .leading, spacing: 12):**

#### Card 4: Spend by Category Chart (EXP-10)

**Data:** Current-month spend per category, aggregated via `BudgetCalculator.monthlySpend`. Includes all categories with `spend > 0`. Excludes uncategorized spend (no category label). Source: D4-05.

**Layout (VStack, alignment: .leading, spacing: 12):**

Row A — Card title:
- Text "Spend by Category" — `.font(.title2)`, semibold

Row B — Swift Charts `BarMark`:
```swift
Chart(topCategoryData) { item in
    BarMark(
        x: .value("Amount", item.spent),
        y: .value("Category", item.name)
    )
    .foregroundStyle(Color.accentColor)
    .annotation(position: .trailing) {
        Text(item.spent.formattedINR())
            .font(.caption)
            .foregroundStyle(.secondary)
    }
}
.chartXAxis(.hidden)          // hide X-axis; amounts shown as annotations
.chartYAxis {
    AxisMarks(values: .automatic) { _ in
        AxisValueLabel()
            .font(.caption)
    }
}
.frame(height: 220)
.chartScrollableAxes(.vertical)  // scrollable if > 7 categories
```
- Bar orientation: **horizontal** (`x` = amount, `y` = category name) so category names are always readable without rotation
- Sorted: descending by `spent` (highest spend at top)
- Max displayed without scrolling: 7 rows. Categories beyond 7 are accessible via vertical scroll (`.chartScrollableAxes(.vertical)`)
- Annotation: abbreviated ₹ amount to the right of each bar (see "Chart Y-axis ₹ labels" formatting below)
- Bar color: `Color.accentColor` for all bars (single-series chart; no multi-color encoding needed)

**Chart amount formatting (annotation and axes):**
- Use abbreviated format for readability: `≥1,00,000 → "₹Xl"` (lakh), `≥1,000 → "₹Xk"`, `<1,000 → "₹X"`. Implement as an extension on `Decimal` (e.g. `formattedINRCompact()`). Full `formattedINR()` is NOT used for axis/annotation labels — too wide.
- Full `formattedINR()` is used in tooltips/selection overlays only.

**Interactive selection (touch):**
- `.chartGesture` or `.chartOverlay` with a `DragGesture` is NOT implemented in Phase 4 (interaction v1 is read-only). Tap-to-select is a v2 enhancement.
- Charts are display-only in v1 per D4-08 (no interactive selection spec'd).

**Empty state — no categorized spend (D4-07):**
- Replace the Chart with: Text "No spend yet this month." — `.font(.subheadline)`, `.secondary`, `.frame(height: 80)`, centered

**Accessibility:**
- `.accessibilityLabel("Spend by category chart")` on the `Chart` view
- Each `BarMark` gets: `.accessibilityLabel("\(item.name), \(item.spent.formattedINR())")` via `.accessibilityValue`
- Use `.accessibilityChartDescriptorRepresentable` is deferred to v2 (full audio chart). Minimum: `accessibilityLabel` on each mark is required.

#### Card 5: Spend Over Time Chart (EXP-11)

**Data:** Bucketed spend aggregated by a new `SpendOverTimeAggregator` pure helper in `Support/`. Bucketing: Week → daily (7 points), Month → daily (28–31 points), Year → monthly (12 points). Source: D4-04. Aggregator mirrors the `CalendarAggregator` shape: pure static methods, no `@Query`, fed by an already-fetched expense array.

**Layout (VStack, alignment: .leading, spacing: 12):**

Row A — Card title:
- Text "Spend Over Time" — `.font(.title2)`, semibold

Row B — Range segmented control:
- `Picker("Range", selection: $selectedRange) { … }` with `.pickerStyle(.segmented)`
- Options: `"Week"`, `"Month"`, `"Year"` (string labels; use an enum `SpendRange` { week, month, year } for the binding type)
- Default selection: `.month`
- Placement: full-width, `.padding(.horizontal, 0)` (no extra horizontal padding; already inside the card's 16pt padding)

Row C — Swift Charts `LineMark`:
```swift
Chart(bucketedData) { point in
    AreaMark(
        x: .value("Date", point.date),
        y: .value("Spend", point.spent)
    )
    .foregroundStyle(Color.accentColor.opacity(0.15))

    LineMark(
        x: .value("Date", point.date),
        y: .value("Spend", point.spent)
    )
    .foregroundStyle(Color.accentColor)
    .lineStyle(StrokeStyle(lineWidth: 2))
    .symbol(.circle)
    .symbolSize(30)
}
.chartXAxis {
    AxisMarks(values: .automatic) { value in
        AxisGridLine()
        AxisValueLabel(format: xAxisFormat(for: selectedRange))
            .font(.caption)
    }
}
.chartYAxis {
    AxisMarks(position: .leading, values: .automatic(desiredCount: 4)) { value in
        AxisGridLine()
        AxisValueLabel {
            if let d = value.as(Decimal.self) {
                Text(d.formattedINRCompact())
                    .font(.caption)
            }
        }
    }
}
.frame(height: 200)
```

**X-axis date formats by range:**
- `Week`: `"EEE"` (Mon, Tue … Sun) — 3-letter day abbreviation
- `Month`: `"d"` (1, 2 … 31) — day-of-month digit
- `Year`: `"MMM"` (Jan, Feb … Dec) — 3-letter month abbreviation

**Y-axis amount format:** Use abbreviated `formattedINRCompact()` (same helper as Card 4).

**Zero-spend buckets:** Include all buckets in the date range even if spend = 0. The line drops to 0 for zero-spend days/months — do not omit points (omitting creates misleading gaps in the line).

**Empty state — no spend in selected range (D4-07):**
- Replace Chart with: Text "No spend yet for this period." — `.font(.subheadline)`, `.secondary`, `.frame(height: 80)`, centered
- The segmented control (Row B) remains visible so the user can try other ranges

**Accessibility:**
- `.accessibilityLabel("Spend over time chart, \(selectedRange.label) view")` on the `Chart`
- Each `LineMark` point: `.accessibilityLabel("\(point.dateLabel), \(point.spent.formattedINR())")` via `.accessibilityValue`

## Interaction Contracts

### Tab Navigation Reorder (D4-01)

- `RootView.selectedTab` default changes from `0` (Expenses) to `0` (Overview/Home — new tag 0).
- Deep-link `selectedTab = 2` in the `kOpenNoteNotification` handler MUST be updated to `selectedTab = 3` (Notes is now tag 3). Failure to do this breaks all notification deep-links from Phase 3.
- Each tab retains its independent `NavigationStack` — no shared state.

### Quick-Add Expense (OVR-04, D4-06)

- Trigger: Navigation bar trailing `+` button on the Overview screen.
- Presents the **existing** `AddExpenseView` as `.sheet(isPresented:)`.
- No new entry UI. No simplified/stripped version. The full custom decimal keypad sheet from Phase 1 is re-presented verbatim.
- After the sheet dismisses (save or cancel), the Overview auto-refreshes via `@Query` reactivity — no manual refresh needed.

### Pinned Note Navigation (OVR-03)

- Tapping "Open note" button or the note title in Card 3 switches to the Notes tab (`selectedTab = 3`) AND sets `deepLinkNoteID = note.id` on `RootView` — triggering the existing Phase 3 deep-link mechanism to open `EditNoteView` for that note.
- This reuses the exact same deep-link path (`kOpenNoteNotification` plus the `selectedTab = 3` + `deepLinkNoteID` binding pattern from Phase 3) but triggered programmatically rather than from a notification.
- Tapping "Go to Notes" (empty state) switches `selectedTab = 3` only (no note ID).

### "Set a budget" nudge (OVR-01 empty state)

- Tapping "Set a budget" switches `selectedTab = 2` (Budgets tab). No sheet is presented from Overview.
- This is the only cross-tab navigation from Overview in v1.

### Spend Over Time — Range Control

- `Picker` binding: `@State private var selectedRange: SpendRange = .month`.

## Chart Implementation Notes

These are binding constraints for the executor (not suggestions).

1. **Pre-aggregate outside the view.** Do not pass raw `@Query` results directly into `Chart {}`. Compute aggregated arrays in an `@Observable` ViewModel or in `LazyVStack` sub-views that receive the pre-computed data. Source: PITFALLS.md Pitfall 15.

2. **`SpendOverTimeAggregator` helper shape** (mirrors `CalendarAggregator`):
   - Pure `enum SpendOverTimeAggregator` with static methods
   - `static func bucket(expenses: [Expense], range: SpendRange, calendar: Calendar) -> [SpendBucket]`
   - `SpendBucket: Identifiable` with `id: Date`, `date: Date`, `spent: Decimal`
   - No `@Query`, no `@Observable`, no SwiftUI — unit-testable TDD default
   - Lives in `MyHomeApp/Support/SpendOverTimeAggregator.swift`

3. **`Decimal` → `Double` for chart values.** Swift Charts `Value` types require `Plottable` conformance. `Decimal` is not `Plottable`; convert via `NSDecimalNumber(decimal: x).doubleValue` at the aggregation boundary (not inside the chart DSL). Money math remains `Decimal`; only the chart input doubles.

4. **Month boundaries.** Reuse `BudgetCalculator.monthBoundaries(for:)` for the "Month" and "Year" range start/end dates. Do not reimplement date math.

5. **Chart accessibility minimum.** Every `BarMark` and `LineMark` must have `.accessibilityLabel` set. Full `AXChartDescriptorRepresentable` is a v2 enhancement.

6. **`formattedINRCompact()` helper.** New extension on `Decimal` in `Decimal+INR.swift`:
   ```swift
   func formattedINRCompact() -> String {
       let d = NSDecimalNumber(decimal: self).doubleValue
       if d >= 100_000 { return "₹\(Int(d / 100_000))L" }
       if d >= 1_000   { return "₹\(Int(d / 1_000))k" }
       return "₹\(Int(d))"
   }
   ```
   Used exclusively for chart axis labels and bar annotations. Full `formattedINR()` used everywhere else.
