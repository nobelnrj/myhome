# Phase 9 — UI Design Contract

> Visual and interaction contract for Phase 9: SchemaV6 & Accounts Management.
> Generated by gsd-ui-researcher. Native SwiftUI / SwiftData / iOS 17+ — all contracts are
> expressed in SwiftUI terms (system colors, SF Symbols, Dynamic Type, Form/List/NavigationStack).

## Design System

| Property | Value |
|----------|-------|

## Spacing Scale

All values are SwiftUI CGFloat points (1pt = 1px on non-Retina; 2×/3× on Retina — values stay
the same). All spacing is in multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4 | Icon-to-label gap, inline subtitle spacing, tight VStack separators |
| sm | 8 | Intra-section row gap, secondary padding |
| md | 16 | Default horizontal padding, section inner padding (matches `CardStyle` default) |
| lg | 24 | Between major sections in a scrollable form |
| xl | 32 | Large vertical breaks (e.g. above CTAs at the bottom of a form) |
| 2xl | 48 | (reserved — not used in this phase's screens) |
| 3xl | 64 | (reserved — not used in this phase's screens) |

**Exceptions:**
- Minimum tappable target: `frame(minHeight: 44)` on all interactive rows — matches existing
  `ManageCategoriesView` and `CategoryPickerView` patterns.
- Profile header in `SettingsView` uses non-standard 56pt avatar circle — retained from existing
  code; do not change.
- `IconTile` default size is 30pt (existing); account icon tiles in list rows use the same 30pt
  default; account detail header may use 44pt for the larger context.

## Typography

All sizes are SwiftUI text style names (Dynamic Type — not hard-coded points). **Two weights only:
regular (400) and semibold (600). Bold (700) is NOT used in this phase.**

| Role | SwiftUI Text Style | Weight | Notes |
|------|-------------------|--------|-------|
| Body | `.body` | regular (400) | Row titles, picker labels, form field values |
| Label / Subtitle | `.subheadline` | regular (400) | Row subtitles, secondary values, picker secondary rows, inline error messages, "as-of date" label below balance |
| Heading / Section | `.body` | semibold (600) | Section-header labels, type labels, balance section headings in AccountDetailView |
| Display / Large | `.largeTitle` | semibold (600) | Balance amount in AccountDetailView (mirrors AddExpenseView amount display) |
| Navigation / Toolbar | `.body` | regular (400) | NavigationTitle inline; toolbar buttons |

> **Revision note (checker fix):** `.title3` was removed — `.body` (semibold) now covers section-
> header and type-label roles. `.caption` was merged into `.subheadline` — `.subheadline` covers
> both secondary subtitles and short error/as-of-date text; `.subheadline` is legible enough for
> inline error messages in a finance context (slightly larger than caption, still clearly secondary).
> `.bold` (700) was dropped; `.largeTitle` balance amount now uses `.semibold` (600). This brings
> the spec to exactly 4 distinct text styles and 2 weights.

**Constraints carried from codebase:**
- `.navigationBarTitleDisplayMode(.inline)` on all new screens — matches every existing screen.
- Toolbar buttons use `.tint(.accentColor)` for affirmative actions; `.tint(Color(.systemRed))` for
  destructive toolbar actions (matches `CategoryPickerView` "Clear" button).
- Plain `Text(value)` always — never `Text(AttributedString(markdown:))` on user input (T-02-15).

## Color

The app uses iOS system colors exclusively — no custom hex palette. Light/dark adaptation is
automatic. All colors below are iOS semantic / system colors expressed as `Color(...)`.

| Role | iOS Color | SwiftUI Expression | Usage |
|------|-----------|-------------------|-------|
| Dominant (60%) | System Background | `Color(.systemBackground)` | NavigationStack bg, List bg |
| Secondary (30%) | Secondary System Background | `Color(.secondarySystemBackground)` | Cards (`.cardStyle()`), grouped List sections, amount display section in forms |
| Accent (10%) | Accent (app tint) | `Color.accentColor` | Save/confirm buttons, checkmark in pickers, "Done" buttons |
| Destructive | System Red | `Color(.systemRed)` | Delete account, archive actions, CC balance owed (negative), "Clear" toolbar button |
| Positive Balance | System Green | `Color(.systemGreen)` | Savings/current positive balance, credit income amounts |
| Warning / Needs Attention | System Orange | `Color(.systemOrange)` | Migration review badge dot |
| Secondary Text | Secondary Label | `.foregroundStyle(.secondary)` | Subtitles, placeholders, chevron indicators |
| Tertiary | Tertiary Label | `.foregroundStyle(.tertiary)` | Chevron in Settings rows (matches `SettingsView` existing pattern) |

**Accent reserved for:**
- Save / confirm CTA buttons in toolbar (`ToolbarItem(.confirmationAction)`)
- "Done" buttons in inline-add rows
- Checkmark (selection indicator) in picker lists
- `Toggle` tint (system default, `.accentColor`)

**Credit-card balance color contract (D-09):**
- CC balance is negative (amount owed). Display using `Color(.systemRed)` with a leading minus
  sign — identical convention to negative expense amounts in `ExpenseRow`.
- Savings/current positive balance: `Color(.systemGreen)`.
- Zero balance: `.primary` (no tint).
- Color is never the sole differentiator — the sign character (+/−) is always present alongside.

**Account icon tile color:**
- User selects from a fixed palette of iOS system colors. Recommended palette (12 swatches):
  `systemBlue`, `systemGreen`, `systemOrange`, `systemRed`, `systemYellow`, `systemIndigo`,
  `systemTeal`, `systemPink`, `systemPurple`, `systemBrown`, `systemCyan`, `systemMint`.
  Stored as `colorHex: String?` (hex encoding) per CloudKit rule 8. Displayed via `IconTile`.

## Screen 1: AccountsListView

**Purpose:** CRUD list for bank accounts under Settings. Mirrors `ManageCategoriesView` closely.

**Navigation:** `SettingsView` → `NavigationLink` → `AccountsListView` (push, not sheet).
Uses `NavigationStack` owned by `SettingsView`'s existing stack.

**Focal point:** The active account's name and balance in the first row of the Active section.
When there are no accounts, the empty state prompt centers itself as the sole visual element.

**Layout:**
```
NavigationBar: "Accounts" (inline) | trailing: "+" button (Add Account)
─────────────────────────────────────────────────────────
List (.insetGrouped)
  Section "Active"
    [AccountRowView] × N  (sorted by sortOrder ascending)
      swipe-left: Archive (orange, "archivebox"), Delete (red, "trash")
      tap → AccountDetailView (NavigationLink push)
  Section "Archived" (collapsed by default, D-08)
    DisclosureGroup("Archived (\(count))")
      [AccountRowView] × M  (greyed out — .opacity(0.5), no swipe-delete)
        swipe-left: Unarchive (blue, "arrow.uturn.left")
─────────────────────────────────────────────────────────
```

**AccountRowView anatomy:**
```
HStack(spacing: 16)                          // md token — gives IconTile + label room to breathe
  IconTile(symbol: account.symbolName, color: accountColor, size: 30)
  VStack(alignment: .leading, spacing: 0)    // no gap between name and type label
    Text(account.name)         .font(.body)       .foregroundStyle(.primary)
    Text(typeLabel)            .font(.subheadline) .foregroundStyle(.secondary)
  Spacer(minLength: 8)
  Text(formattedBalance)       .font(.body)
    .foregroundStyle(balanceColor)  // systemRed / systemGreen / .primary
  Image(systemName: "chevron.right") .font(.subheadline.weight(.semibold)) .foregroundStyle(.tertiary)
frame(minHeight: 44)
```

> **Revision note (checker fix — Spacing BLOCKING 3):**
> - `HStack(spacing: 12)` corrected to `spacing: 16` (md token). 12 is off-scale.
> - `VStack(spacing: 1)` corrected to `spacing: 0`. 1 is off-scale; 0 is the correct value for
>   visually flush name/type label stacking.

**Empty state (no accounts):**
- `ContentUnavailableView("No Accounts Yet", systemImage: "building.columns", description: Text("Tap + to add your first bank account."))`

**Migration review badge:**
- When `UserDefaults.accountReviewPending == true`: show an orange badge dot on the Settings row
  for "Accounts" (not a modal interrupt). The badge is a `Circle().fill(Color(.systemOrange))` of
  size 8pt overlaid top-right on the `IconTile`. Tapping the "Accounts" row opens `AccountsListView`
  which contains the review prompt as a pinned Section at the top (see Screen 5).

**Toolbar:**
- `ToolbarItem(.primaryAction)`: `Image(systemName: "plus")` → presents `EditAccountView` as sheet.
  `.accessibilityLabel("Add Account")`.

## Screen 2: EditAccountView (Create / Edit)

**Purpose:** Form to create a new account or edit an existing one. Presented as a sheet with its
own `NavigationStack` (NavigationStack-in-sheet pattern — matches `CategoryPickerView`).

**Navigation title:** "New Account" (create) / "Edit Account" (edit). `.inline`.

**Toolbar:**
- Leading: `Button("Cancel")` → dismiss without saving.
  > **Copywriting convention note (checker fix — BLOCKING 1):** "Cancel" is the codebase-wide
  > iOS `.cancellationAction` label. `CategoryPickerView` uses `Button("Cancel")` on
  > `.cancellationAction` placement (confirmed: `CategoryPickerView.swift` line 83). This is a
  > locked, app-wide iOS idiom — the checker should treat this as a codebase-established convention,
  > not a generic blocklisted word.
- Trailing: `Button("Save Account")` → save + dismiss. `.tint(.accentColor)`. Disabled until name
  is non-empty.

**Form layout (List `.insetGrouped`):**

```
Section 1 — Identity
  Row: Icon + Color picker row
    HStack
      IconTile(symbol, color, size: 44)   (tappable → icon/color picker inline or sub-sheet)
      VStack
        Text("Icon & Color").font(.body)
        Text(selectedSymbolLabel).font(.subheadline).foregroundStyle(.secondary)
    frame(minHeight: 44)

  Row: Name TextField
    TextField("Account name", text: $name)
    .font(.body)
    .frame(minHeight: 44)

  Row: Last 4 TextField (optional, numeric)
    TextField("Last 4 digits (optional)", text: $last4)
    .keyboardType(.numberPad)
    .font(.body)
    .frame(minHeight: 44)

Section 2 — Type
  Picker("Account Type", selection: $typeRaw)
    .pickerStyle(.segmented)
    Labels: "Savings" | "Current" | "Credit Card"

Section 3 — Opening Balance
  (shown only when creating, or always when editing)
  Row: Balance amount
    HStack
      Text(typeRaw == "credit_card" ? "Amount Owed" : "Opening Balance")
        .font(.body)
      Spacer()
      TextField("0", value: $balanceBaseline, format: .number)
        .keyboardType(.decimalPad)
        .multilineTextAlignment(.trailing)
        .font(.body)
        .foregroundStyle(typeRaw == "credit_card" ? Color(.systemRed) : .primary)
    frame(minHeight: 44)

  Row: As-of date
    DatePicker("As of", selection: $balanceAsOfDate, displayedComponents: [.date])
      .font(.body)
    (defaults to today — D-10)

Section 4 — (edit mode only) Danger zone
  Button("Archive Account") { ... }
    .foregroundStyle(Color(.systemOrange))
    frame(minHeight: 44)
```

**Inline error display:**
- Name empty → `.subheadline` below name field in `Color(.systemRed)`: "Account name cannot be empty."
- Name duplicate → `.subheadline`: "An account with that name already exists."
- Balance out of range → `.subheadline`: "Balance must be less than ₹1,00,00,00,000."

> **Revision note:** Error text uses `.subheadline` (regular) instead of `.caption`, consistent
> with the 4-style typography table. `.subheadline` is legible for error messages in a finance app.

**Icon + Color picker (reuse ManageCategoriesView / ACCT-03):**
- Presented as an inline expandable section within the form OR a separate sheet depending on the
  planner's preference; the contract requires both a symbol picker and a color swatch grid.
- Symbol picker: scrollable grid of SF Symbol names relevant to banking/finance. Recommended set
  (20+ symbols): `building.columns`, `creditcard`, `banknote`, `wallet.pass`, `dollarsign.circle`,
  `house`, `car`, `briefcase`, `stethoscope`, `cart`, `fork.knife`, `bolt`, `fuelpump`, `bag`,
  `graduationcap`, `airplane`, `gift`, `heart`, `tag`, `star`.
- Color swatch grid: 12 swatches from the account palette defined in the Color section above.
  Selected swatch gets a `checkmark` overlay in white.
- Both selections confirmed immediately on tap (no separate "Apply" button).

## Screen 3: AccountDetailView

**Purpose:** Per-account balance headline + attributed expense list (ACCT-06, D-07). Read-only
balance; edit reachable via toolbar button.

**Navigation:** Push from `AccountsListView` row tap. Uses existing `NavigationStack`.

**Navigation title:** `account.name ?? "Account"` (inline).

**Toolbar:**
- Trailing: `Button("Edit")` → presents `EditAccountView` as sheet.

**Layout:**

```
ScrollView (or List with sticky header)
  ─────────────────────────────────────────────────────────
  Balance Card  (.cardStyle(), padding: 16)
    VStack(alignment: .leading, spacing: 4)
      Text(typeLabel)  .font(.subheadline) .foregroundStyle(.secondary)
      Text(formattedBalance)  .font(.largeTitle.weight(.semibold))
        .foregroundStyle(balanceColor)  // red/green/.primary
      HStack
        IconTile(symbol: account.symbolName, color: accountColor, size: 20)
        Text("as of \(asOfDateFormatted)")  .font(.subheadline) .foregroundStyle(.secondary)
  ─────────────────────────────────────────────────────────
  (same List structure as ExpenseListView, filtered to this account)
  Section header per day: date string + day total (same pattern as ExpenseListView)
  ExpenseRow(expense:) for each attributed expense
  ─────────────────────────────────────────────────────────
```

> **Revision note:** Balance amount changed from `.largeTitle.weight(.bold)` to
> `.largeTitle.weight(.semibold)` — aligns with the 2-weight constraint (regular + semibold only).
> "as of" text changed from `.caption` to `.subheadline` — consistent with the 4-style table.

**Empty state (no attributed expenses):**
- `ContentUnavailableView("No Expenses", systemImage: "tray", description: Text("Expenses attributed to this account will appear here."))`

**Balance display contract:**
- `formattedBalance` uses `.currency(code: "INR").locale(Locale(identifier: "en_IN"))` — same
  `formattedINR()` extension used throughout the app.
- `balanceColor`:
  - CC: always `Color(.systemRed)` (amount owed is negative per D-09).
  - Savings/current positive: `Color(.systemGreen)`.
  - Savings/current zero: `.primary`.
  - Savings/current negative (overdrawn): `Color(.systemRed)`.
- Balance updates without manual refresh — computed from `@Query` results (D-10, ACCT-05).

- Trailing: `Button("Clear")` (`.tint(Color(.systemRed))`, `.destructiveAction` placement) — shown

**List layout (`.insetGrouped`):**

## Screen 5: Migration Review Sheet (D-02)

**Purpose:** After V5→V6 migration, surfaces auto-created accounts for rename / type-fix / delete
before they are treated as permanent. Shown as a sheet triggered from `AccountsListView` when
`UserDefaults.accountReviewPending == true`.

**Decision (open question 2 from RESEARCH.md):** This is a **Settings-row badge + sheet**, not a
RootView modal. The badge on the "Accounts" Settings row alerts the user; they navigate to
`AccountsListView` which shows the review section pinned at the top. Less disruptive — user can
ignore until they want to review. Source: RESEARCH.md open question 2 recommendation.

**Review section in AccountsListView (pinned above Active section):**
```
Section {
  HStack
    Image(systemName: "exclamationmark.circle.fill")  .foregroundStyle(Color(.systemOrange))
    VStack(alignment: .leading, spacing: 4)            // xs token — replaces off-scale spacing: 2
      Text("Review Auto-Created Accounts")  .font(.body.weight(.semibold))
      Text("These accounts were created from your transaction history. Rename, change type, or delete before they're finalized.")
        .font(.subheadline)  .foregroundStyle(.secondary)
  Button("Review Now →")  .font(.body)  .tint(.accentColor)
    → presents MigrationReviewSheet as .sheet
}
```

> **Revision note (checker fix — Spacing BLOCKING 3 + Typography):**
> - `VStack(spacing: 2)` corrected to `spacing: 4` (xs token). 2 is off-scale.
> - Banner body text changed from `.caption` to `.subheadline` — consistent with 4-style table.

**MigrationReviewSheet layout (sheet, NavigationStack-in-sheet):**
- Title: "Review Accounts" (inline).
- Toolbar trailing: `Button("Done")` → dismiss + clear `accountReviewPending` flag.
  > **Copywriting convention note (non-blocking fix):** "Done" is the iOS `.confirmationAction`
  > system idiom used app-wide for sheet dismissal after completing a review task. It is a locked
  > iOS platform convention, not a generic label — the checker should treat it as an established
  > codebase idiom.
- List `.insetGrouped`:
  - One row per auto-created account.
  - Inline rename: tap name → editable TextField (same pattern as `ManageCategoriesView`).
  - Type picker: segmented "Savings / Current / Credit Card" inline below name.
  - Swipe-to-delete with `confirmationDialog`.
- Footer note (Section footer text): "Changes here are permanent. Tap Done when finished."

## Screen 6: ExpenseListView — Account Filter Addition

**Entry point:** Existing `ExpenseListView` toolbar leading menu.

**Change:** Add an "Account" dimension to the filter toolbar menu, alongside the existing
Category filter. The filter menu becomes a two-picker menu (Category + Account), or the account
filter is a second `Menu` button in the toolbar leading area.

**Recommended approach:** Single `Menu` with two `Picker` sections (Category then Account), one
filled-icon indicator when either filter is active. Keeps the toolbar uncluttered.

**AccountFilter enum (mirrors CategoryFilter):**
```swift
private enum AccountFilter: Hashable {
    case all         // label: "All Accounts",  systemImage: "building.columns"
    case unassigned  // label: "Unassigned",    systemImage: "questionmark.circle"
    case account(UUID)
}
```

**Filter indicator:** toolbar icon becomes `.fill` variant when any non-`.all` filter is active
(same pattern as the existing category filter indicator).

**Filter label in empty state:** "No expenses for [category] · [account]."

## Interaction Contracts

### Touch Targets

All interactive rows: `frame(minHeight: 44)` — mandatory, no exception.

### Destructive Actions

| Action | Trigger | Confirmation |
|--------|---------|-------------|
| Delete account | Swipe-left "Delete" button on AccountsListView row | `confirmationDialog("Delete Account?", titleVisibility: .visible)` with destructive `Button("Delete Account", role: .destructive)` + `Button("Cancel", role: .cancel)`. Message: "All expenses attributed to this account will become unassigned. This cannot be undone." |
| Archive account | Swipe-left "Archive" on AccountsListView row OR "Archive Account" button in EditAccountView | No dialog — archive is reversible. Immediate action. |
| Unarchive account | Swipe-left "Unarchive" on archived row | No dialog — non-destructive. Immediate action. |
| Delete auto-created account in review | Swipe-to-delete in MigrationReviewSheet | `confirmationDialog` same as delete above. |

### Navigation Flows

```
SettingsView
  ├─ NavigationLink "Accounts" → AccountsListView (push)
  │     ├─ NavigationLink row tap → AccountDetailView (push)
  │     │     └─ Toolbar "Edit" → EditAccountView (sheet)
  │     ├─ Toolbar "+" → EditAccountView (sheet, create mode)
  │     └─ Review banner "Review Now" → MigrationReviewSheet (sheet)
  │
ExpenseListView
  └─ Toolbar filter menu → AccountFilter picker (inline in Menu)
  
AddExpenseView / EditExpenseView
  └─ "Account" row tap → AccountPickerView (sheet)
```

### Balance Reactivity

Balance displayed in `AccountDetailView` and `AccountRowView` reacts automatically to expense
changes via `@Query`. No "Refresh" button or pull-to-refresh. This is a contract requirement —
the executor must not add a manual refresh affordance (D-10, ACCT-05).

### Archived Section Collapse

The "Archived" section in `AccountsListView` uses `DisclosureGroup`. Initial state: collapsed
(`isExpanded = false`). State is NOT persisted — collapses back on each view appearance. This
matches the low-frequency nature of archived account access.

## Accessibility Contracts

- All `IconTile` instances: `.accessibilityHidden(true)` (decorative — carried from existing pattern).
- All interactive rows: `frame(minHeight: 44)` for touch target.
- Expense rows: `.accessibilityElement(children: .combine)` (carried from existing `ExpenseRow`).

- Sign toggle in add form (if added): `.accessibilityLabel("Toggle sign")` (carried from `AddExpenseView`).
