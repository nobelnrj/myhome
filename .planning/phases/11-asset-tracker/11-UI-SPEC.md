# Phase 11 — UI Design Contract: Asset Tracker

> Visual and interaction contract for the Asset Tracker feature. Generated by gsd-ui-researcher,
> verified by gsd-ui-checker.
>
> This is a SwiftUI / iOS app with a settled design language — NOT a web app. The
> "design system" is the app's own shared components (CardStyle, IconTile, DonutChart, etc.)
> and iOS system semantics (adaptive colors, SF Symbols, system fonts). No shadcn, no Tailwind.

## Color

All colors are iOS adaptive semantic colors — they respond correctly to Light/Dark mode and
Dynamic Type without any override. No hex values are hardcoded for semantic roles.

| Role | Value | Usage |
|------|-------|-------|
| Dominant surface (60%) | `Color(.systemGroupedBackground)` | Full-screen background for all list/scroll views |
| Secondary surface (30%) | `Color(.secondarySystemBackground)` | Card backgrounds via `cardStyle()`, form cells |
| Accent (10%) | `Color.accentColor` (system blue by default) | See reserved list below |
| Positive gain | `Color(.systemGreen)` | Absolute / % gain when positive; savings balance when positive |
| Negative / loss | `Color(.systemRed)` | Absolute / % gain when negative; staleness badge dot; destructive actions |
| Warning / stale | `Color(.systemOrange)` | Staleness badge when `navAsOfDate` is > 1 calendar day old |
| Tertiary text | `.tertiary` foreground style | Chevron icons, "/ budget" denominators, secondary metadata |
| Destructive | `Color(.systemRed)` | Delete confirmations and delete button labels only |

**Accent reserved for:**
- "Save Asset" / "Save" toolbar confirmation button (`.tint(.accentColor)`)
- "Fetch now" button in AMFISchemePickerView when no schemes are loaded
- Pull-to-refresh tint
- Active segment highlight in range pickers
- "See all" / "See holdings" navigation text links (sectionHeader action buttons)
- Selected AMFI scheme row highlight background (`.accentColor.opacity(0.12)`)
- Selected icon in EditAssetView icon picker (`.accentColor.opacity(0.15)` background + `.accentColor` tint, mirrors EditAccountView)

**Donut segment colors (asset-class mapping):**

| Asset Class | Color | Rationale |
|-------------|-------|-----------|
| Mutual Funds | `Color(.systemBlue)` | Consistent with accounts (blue = financial instrument) |
| Stocks | `Color(.systemGreen)` | Equity growth association |
| NPS | `Color(.systemOrange)` | Pension / long-term |
| Cash | `Color(.systemTeal)` | Liquid / safe |

Cash segment value passed to `DonutChart`: `max(cashNetBalance, Decimal(0))` — clamped to zero when
CC debt exceeds savings (negative `cashValue` crashes `SectorMark`). The displayed center label
shows the true (possibly negative) net-worth total via `Text`.

### Surface 1 — Overview Net-Worth Card (D-04)

**Location:** `OverviewView` / `OverviewMonthContent` scroll body, inserted after the spend hero
card and before "Where it's going".

**Section header:** "Net Worth" — follows the existing `sectionHeader(_:action:)` pattern with
action `("See holdings", { /* navigate to AssetsListView */ })`.

**Card layout** (`cardStyle(cornerRadius: 16, padding: 18)`):

```
┌──────────────────────────────────────────────────┐
│  [DonutChart 132×132]    NET WORTH (caption2)     │
│       center:            ₹X,XX,XXX (headline)    │
│       "NET WORTH"        ─────────────────        │
│       ₹X,XX,XXX          MF     ₹X,XX,XXX (sub)  │

│                          NPS    ₹X,XX,XXX (sub)  │
│                          Cash   ₹X,XX,XXX (sub)  │
└──────────────────────────────────────────────────┘
```

- HStack layout mirroring `WhereItsGoingCard` (donut left, legend VStack right)

**Negative net-worth display:** Center label in donut shows `totalNetWorth.formattedINRWhole()`
including negative sign (e.g., "−₹50,000"). The donut renders with the cash slice clamped at 0.

**Structure:** Mirrors `AccountsListView` exactly:

**Holding row layout** (`frame(minHeight: 44)`):

- Line 1: `Text(asset.name ?? "—").font(.body).foregroundStyle(.primary).lineLimit(1)`

### Surface 3 — Edit Asset Sheet (Add / Edit) (D-01 / D-02 / ASSET-01 / ASSET-02 / ASSET-04)

**Presentation:** `NavigationStack`-in-sheet (mirrors `EditAccountView`). `nil` asset = create;

**Section 1 — Identity:**
- Asset name `TextField("Holding name", text: $name)` — `.body`, `minHeight: 44`
- Asset class `Picker` segmented: "Mutual Fund" / "Stock" / "NPS"
  - Segmented control mirrors account type picker in `EditAccountView`

**Section 2 — Mutual Fund scheme (conditional — only when `assetClass == .mutual_fund`):**

- Display-only computed row: "Total cost" = `units × costBasisPerUnit`, formatted `.formattedINRWhole()`
  in `.secondary`

**Section 4 — Current NAV / Price:**
- For `mutual_fund`: label "Current NAV (auto-fetched)" — field is editable for manual override
- For `stock` / `nps`: label "Current price" — field is primary entry method, no auto-fetch hint

- "Delete Holding" button in `.systemRed` — triggers confirmation dialog, not inline

- `.cancellationAction`: "Cancel"

### Surface 4 — AMFI Scheme Picker (D-01 / D-02)

**Presentation:** Navigation destination within the `EditAssetView` NavigationStack
(not a separate sheet). Pushed when user taps the scheme row in Section 2.

**Header:** Navigation title "Choose Scheme", `.inline` display mode.

**States:**

**State A — Schemes loaded (post-first-fetch):**
- `List` with `searchable` modifier (`.searchable(text: $query, placement: .navigationBarDrawer(displayMode: .always))`)
- Each row: scheme name (`.body`, 2 lines max) + scheme code (`.caption`, `.secondary`)
- Selected scheme: row background tinted `Color.accentColor.opacity(0.12)` + trailing checkmark
  `Image(systemName: "checkmark").foregroundStyle(.accentColor)`
- Tap row: sets `amfiSchemeCode` on the parent form state, pops back
- Search filters by scheme name prefix/contains (case-insensitive). Show all ~13,000+ entries when
  query is empty (List virtualizes automatically — no performance concern)

**State B — Pre-first-fetch (no schemes loaded yet):**
```
ContentUnavailableView(
    "No Schemes Loaded",
    systemImage: "arrow.down.circle",
    description: Text("Scheme data hasn't been fetched yet.")
)
```
Below the ContentUnavailableView: a `Button("Fetch Now")` — `.tint(.accentColor)` — that calls
`amfiNavService.forceRefresh()` (bypasses daily gate; fetches immediately).

**State C — Fetching in progress:** `ProgressView("Loading schemes…")` centered in the list area.
Replaces rows while fetch is in flight. Never blocks the rest of the form.

**State D — Fetch failed (offline):** Re-show State B with description "Couldn't load schemes.
Check your connection and try again." + "Try Again" button.

### Surface 6 — Net-Worth Trend Chart (ASSET-08)

**Location:** Embedded inside the Overview Net-Worth Card as a secondary row below the donut +
legend, OR shown as a full section when tapping into `AssetsListView` (implementation decision:
planner picks placement; this contract covers both options).

**Chart specification** (mirrors `SpendOverTimeChart` pattern exactly):

```swift
Chart(snapshots) { snap in
    AreaMark(x: .value("Date", snap.date),
             y: .value("Net Worth", Double(snap.totalNetWorth as NSDecimalNumber)))
        .foregroundStyle(Color.accentColor.opacity(0.15))
    LineMark(x: .value("Date", snap.date),
             y: .value("Net Worth", Double(snap.totalNetWorth as NSDecimalNumber)))
        .foregroundStyle(Color.accentColor)
        .lineStyle(StrokeStyle(lineWidth: 2))
}
.chartYAxis {
    AxisMarks(position: .leading, values: .automatic(desiredCount: 4)) { value in
        AxisGridLine()
        AxisValueLabel { if let d = value.as(Double.self) { Text(Decimal(d).formattedINRCompact()).font(.caption) } }
    }
}
.chartXAxis {
    AxisMarks(values: .automatic) { value in
        AxisGridLine()
        AxisValueLabel(format: .dateTime.month(.abbreviated))
            .font(.caption)
    }
}
.frame(height: 140)
```

- Decimal → Double at boundary: `NSDecimalNumber(decimal: snap.totalNetWorth).doubleValue`
  (never pass `Decimal` directly to `.value(...)`)
- Y-axis labels: `formattedINRCompact()` (e.g. "₹5L", "₹50k")
- X-axis: abbreviated month names (always month granularity — snapshots are daily but labels
  show months to avoid crowding)
- Background: none (chart rendered inside existing card via `cardStyle`)
- Empty state (no snapshots): `Text("No history yet.").font(.subheadline).foregroundStyle(.secondary)` centered, `frame(height: 80)` (mirrors SpendOverTimeChart empty state)
- Accessibility: `.accessibilityLabel("Net worth trend chart")`; chart itself `.accessibilityHidden(true)` (mirrors DonutChart)

### Surface 7 — Staleness Badge Component (ASSET-09)

**Reusable component:** `StalenessView` (or `StaleBadge`) — small inline indicator.

**Staleness rule:** `navAsOfDate` is stale when `Calendar.current.dateComponents([.day], from: navAsOfDate, to: startOfTodayIST).day ?? 0 > 1` (D-10: calendar-day threshold, ignores weekends/holidays).

**Fresh state (not stale):** nothing rendered (`.hidden` or `EmptyView`)

**Stale state:**
```
[Image(systemName: "clock.badge.exclamationmark").foregroundStyle(Color(.systemOrange))]
" Stale"
```
- Font: `.caption`
- Color: `Color(.systemOrange)`
- Accessibility label: "Price is stale. Last updated \(navAsOfDate?.formatted(…) ?? "unknown")."

**As-of date label** (shown on all surfaces regardless of freshness):
- Format: `"as of \(navAsOfDate.formatted(.dateTime.day().month(.abbreviated).year()))"` →
  `"as of 10-Jun-2026"`
- Font: `.subheadline`, color: `.secondary`
- When `navAsOfDate == nil` (no NAV set yet): show `"price not set"` in `.secondary`

## Interaction Contracts

### Pull-to-Refresh

`AssetsListView` supports `.refreshable { amfiNavService.forceRefresh() }`. This bypasses the daily
IST gate and triggers an immediate AMFI fetch. The standard iOS refresh indicator is used; no
custom spinner. On completion (success or failure), the refresh indicator dismisses automatically.
The existing staleness badge updates reactively via `@Query`.

### Fetch Failure (D-07)

On AMFI fetch failure (offline / server error): fail silently. Do NOT show an error alert or banner.
The staleness badge on stale holdings communicates the data age. The pull-to-refresh indicator
completes normally (no error state in the refresh indicator itself).

### Tapping the Overview Net-Worth Card (D-05)

The entire card body is wrapped in a `Button` (`.buttonStyle(.plain)`) that pushes `AssetsListView`
via the `NavigationStack` already present in `OverviewView`. Alternatively the section header's
"See holdings" action navigates the same route. Both entry points push the same view.

### Scheme Picker Search

`searchable` text field filters the scheme list as the user types. Search is client-side on the
in-memory `[AMFIScheme]` array (no network call per keystroke). Minimum 1 character triggers filter.
Empty query shows all schemes. Picker does NOT auto-select on unique match — user must tap a row.

### Zero Cost Basis Gain/Loss

When `costBasisPerUnit` is nil or 0: percentage gain is `nil`. The gain row in `AssetDetailView`
shows the absolute gain (which is equal to current value) and "—" for the percentage column. This
is not an error state — render identically to a normal gain row, just with "—" in the % slot.

### Negative Net-Worth Donut

When `cashValue` (net account balances) is negative and exceeds the sum of holdings (total net
worth is negative): the donut renders with the cash segment clamped to 0 (prevents SectorMark
crash). The center label still shows the true negative total (e.g. "−₹50,000"). The donut appears
without a cash slice, showing only the positive asset classes. This is the correct visual for a
household carrying more CC debt than savings + investments.
